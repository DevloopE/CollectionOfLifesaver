第 21 章 数值方法

本书前面 20 章的努力可以总结成一句话：我们学会了一大堆方法，每种方法只能解一类很狭窄的方程——可分离变量的、线性的一阶或二阶常系数的、用拉普拉斯能弄掉的、能用幂级数硬展开的，等等。每一类技巧都很漂亮，每一类技巧背后都有一段历史。但你可能已经隐约察觉到一件让人沮丧的事：在所有可能的微分方程构成的"汪洋大海"里，能用以上任何一种闭式技巧解出来的方程，是一座极小的孤岛。绝大多数实际问题，写出方程后没有人——包括你将来的导师、你的导师的导师——能用初等函数把解 \(y(t)\) 写出来。

这是个坏消息吗？只在你坚持要"闭式表达式"的情况下是。如果你愿意接受"在每一个我关心的时刻 \(t\)，告诉我 \(y(t)\) 的一个非常接近真值的数"——也就是接受一张数值表，或者一条用计算机画出的曲线——那么世界就完全敞开了。这就是数值方法要做的事。本章我们要造一台不依赖任何闭式技巧的机器：你给它一个微分方程和初始条件，它给你一张 \(t \mapsto y(t)\) 的数值表，要多准有多准（代价是更多的计算）。

本章我们将做以下事情：

  • 说清楚为什么数值方法是绕不开的（21.1）。
  • 把"用切线沿着方向场往前爬"这个最朴素的想法系统化，得到 Euler 法（21.2-21.3）。
  • 分析 Euler 法的误差，理解"精度"和"步长"之间的权衡（21.4）。
  • 用一个小升级——用斜率的"平均"而不是单点斜率——得到改进 Euler 法（21.5）。
  • 给出工业界几乎所有 ODE 求解器的"瑞士军刀"——经典 RK4 法的完整公式（21.6）。
  • 讨论"刚性"方程为什么会让显式方法崩溃，以及隐式 Euler 怎么救场（21.7）。
  • 聊聊自适应步长是怎么自动决定 \(h\) 的（21.8）。
  • 把所有这些方法在同一道例题上跑一遍，肉眼对比精度（21.9）。
  • 给出一段 Python/MATLAB 风格的伪码，让你回家就能动手（21.10）。

这一章是计算方向的，但目标不是把你培养成数值分析专家——那是另外一门课。目标是让你知道在什么场合该掏出哪种方法，并且当你看到别人写"用 RK4 解了这个 ODE"时，你能在脑子里完整复盘那一行命令背后到底发生了什么。

21.1 为什么需要数值方法

先看一个具体的方程：

\[ y' = \sin(t \, y), \qquad y(0) = 1. \]

请试着用任何前面学过的方法把它解出来。可分离变量？不行，右边是 \(t\) 和 \(y\) 缠在一个 \(\sin\) 里面，分不开。线性的？不是，它对 \(y\) 是非线性的。Bernoulli 的某种推广？不是。能找到积分因子？右边不是 \(P(t)y + Q(t)\) 的形式，根本谈不上积分因子。能用拉普拉斯？拉普拉斯只对线性方程有用。能用幂级数？理论上可以，但 \(\sin(ty)\) 展开之后会出现 \(y, y^3, y^5, \ldots\) 的混合，递推关系迅速失控。

结论：这个方程没有闭式解。但它一点都不"病态"——右边 \(\sin(ty)\) 是处处光滑的，初值 \(y(0) = 1\) 也再普通不过，按照附录 A 的存在唯一性定理，解在 \(t = 0\) 的某个邻域里存在且唯一。所以"解"是确实存在的，只不过这个解不是任何"我们知道名字"的函数。

这种情况比你以为的更常见。事实上，反过来才罕见——能写成闭式的方程少得可怜，而且大都是工程师或物理学家精心调整出来的"漂亮模型"。一旦你跨过把现实简化成方程的那道门槛，你面对的几乎一定是没有闭式解的方程。一些你已经熟悉的例子：

  • 非线性单摆方程：\(\theta'' + \frac{g}{L}\sin\theta = 0\)。我们在第 12 章用线性化处理过它（小角度近似 \(\sin\theta \approx \theta\)），但角度大的时候这个近似就崩了，原方程没有闭式解（它的"解"是椭圆函数，已经超出初等函数）。
  • 三体问题：在万有引力下三个质点如何运动。Poincaré 在 19 世纪末证明这个方程没有像两体问题那样的闭式解。但所有星历表都是数值积分出来的，准到我们能把宇宙飞船送到火星表面。
  • SIR 流行病模型（22.3 节会详谈）：易感、感染、康复人群之间的非线性方程组，没有闭式解，但每一份新闻里出现的"感染曲线"图都是数值解。
  • Navier–Stokes 方程：流体力学的核心方程组，是不是有光滑解都还是 Clay 学院的百万美金问题，更别说闭式解。但所有飞机机翼、所有汽车空气动力学，都是数值解出来的。

所以，"找闭式解"是一种奢侈。当奢侈品没有的时候，我们用数值解。本章其余部分都在讲这件事怎么干。

顺便提一句：数值方法不是"次等品"。从应用的角度，一张精度足够高的数值表，跟闭式公式给出的信息是等价的——你能算出任意时刻的 \(y\)，能画出图像，能求极值，能做后续优化。它唯一比不上闭式公式的地方是：你没法对它做符号运算（比如取极限 \(t \to \infty\) 看渐近行为，或者求关于参数的偏导）。所以，理想的工作流程是：先用解析方法（如果有的话）把结构看清楚——平衡点、稳定性、长期行为——然后用数值方法在具体场合把曲线画出来。这两件事互补，谁也不能完全代替谁。

21.2 离散化思想

让我们退一步，问一个更基本的问题：什么叫"知道一个函数 \(y(t)\)"？

从纯数学的角度，知道 \(y\) 意味着对每一个实数 \(t\)，你都能给出一个实数 \(y(t)\)。但实数有不可数多个，计算机用有限位数表示数，根本不可能存下所有时刻的 \(y\) 值。所以从计算机的角度，"知道 \(y\)"必然变成"知道 \(y\) 在有限多个时刻 \(t_0, t_1, t_2, \ldots, t_N\) 上的值"。这叫做离散化。

最自然的离散化是把时间分成等距的小段。设我们关心区间 \([t_0, T]\)，选一个步长 \(h > 0\)，在

\[ t_n = t_0 + n h, \qquad n = 0, 1, 2, \ldots, N \]

这些点上找 \(y\) 的近似值。我们用 \(y_n\) 来代表"在 \(t_n\) 处算法给出的近似值"，而 \(y(t_n)\) 代表"真实解在 \(t_n\) 处的值"——这两个不同，差就是误差。

这套记号很重要，从现在开始要严格区分：

  • \(y(t)\) 是真解（连续时间的函数），你大多数时候不知道它长什么样；
  • \(y_n\) 是数值近似（在格点 \(t_n\) 上的一个数）。

一个数值方法的目的就是给出一个递推关系：知道 \(y_n\)，怎么算出 \(y_{n+1}\)。如果递推关系够准，迭代下去得到的序列 \(\{y_n\}\) 就会"看起来像"真解 \(y(t_n)\)。

怎么得到这个递推？从微分方程本身。方程 \(y' = f(t, y)\) 告诉我们：在每一点 \((t, y)\)，曲线的斜率是 \(f(t, y)\)。所以如果我现在站在 \((t_n, y_n)\) 这一点上，我手里就有一个斜率 \(f(t_n, y_n)\)。如果我沿着这个斜率走很短一段水平距离 \(h\)，那么纵坐标的增量大约就是

\[ \Delta y \approx h \cdot f(t_n, y_n). \]

这不就是切线近似嘛！高中学过：函数 \(y(t)\) 在 \(t_n\) 附近被它的切线很好地近似，

\[ y(t_n + h) \approx y(t_n) + h \cdot y'(t_n) = y(t_n) + h \cdot f(t_n, y(t_n)). \]

把"真值 \(y(t_n)\)"换成"近似 \(y_n\)"，把"真值 \(y(t_n+h)\)"换成"近似 \(y_{n+1}\)"，就得到了我们的第一个数值方法的雏形：

\[ y_{n+1} = y_n + h \cdot f(t_n, y_n). \]

就这么一行。这就是 Euler 法。后面那些更花哨的方法（RK4 之类）本质上都在干同一件事——用更聪明的方式估计"在区间 \([t_n, t_{n+1}]\) 上 \(y\) 的平均斜率应该是多少"——但骨架都是上面这一行：

\[ \boxed{\text{下一个值} = \text{当前值} + h \cdot \text{某种斜率估计}.} \]

把这个骨架记住。整章只有这一个想法，剩下的全是细节。

21.3 Euler 法

把上面那一行作为算法记下来，并起个名字。

显式 Euler 法（forward Euler）： 给定 \(y' = f(t, y)\)，\(y(t_0) = y_0\)，步长 \(h\)。对 \(n = 0, 1, 2, \ldots, N-1\)，依次计算

\[ \boxed{y_{n+1} = y_n + h \cdot f(t_n, y_n), \qquad t_{n+1} = t_n + h.} \]

这个方法是 Euler 在 1768 年给出的，比微分方程作为一门学科还要老。它的几何图像非常干净：站在已知点 \((t_n, y_n)\) 上，朝着方向场指示的方向走一小步水平距离 \(h\)，落到新点 \((t_{n+1}, y_{n+1})\)。然后把这个新点当作新的"已知点"，再走一小步，再走一小步……如果把这些线段连起来画出来，你会得到一条折线，它近似真实的解曲线。步长越小，折线越贴近真解；步长越大，折线越快地偏离真解。

这里值得停下来强调"显式"两字的含义。注意 \(y_{n+1}\) 单独出现在等式左边，等式右边只用到已知量 \(y_n\) 和 \(t_n\)。所以"算下一步"就是一步代数：把数代进去，乘一下，加一下。这种方法叫显式。后面我们会看到隐式方法，那种方法的 \(y_{n+1}\) 同时出现在等式两边，每一步都要解一个方程。

21.3.1 一个动手的例子

来手算一个例子。考虑

\[ y' = y, \qquad y(0) = 1. \]

这个方程当然有闭式解 \(y(t) = e^t\)——我们故意挑了一个有闭式解的方程，这样算完之后好对照精度。取步长 \(h = 0.1\)，目标算到 \(t = 1\)。这里 \(f(t, y) = y\)。

第一步：\(y_0 = 1\) 在 \(t_0 = 0\)。

\[ y_1 = y_0 + h \cdot f(t_0, y_0) = 1 + 0.1 \cdot 1 = 1.1. \]

第二步：\(y_1 = 1.1\) 在 \(t_1 = 0.1\)。

\[ y_2 = y_1 + h \cdot f(t_1, y_1) = 1.1 + 0.1 \cdot 1.1 = 1.21. \]

第三步：\(y_2 = 1.21\) 在 \(t_2 = 0.2\)。

\[ y_3 = 1.21 + 0.1 \cdot 1.21 = 1.331. \]

看出规律了吗？每一步都是把当前值乘以 \(1.1\)。所以 \(y_n = 1.1^n\)。10 步以后到 \(t = 1\)，得到 \(y_{10} = 1.1^{10} \approx 2.5937\)。

真值是 \(e \approx 2.71828\)。误差是 \(2.71828 - 2.59374 \approx 0.1245\)。也就是说，10 步 Euler 给出了大约 4-5% 的相对误差。还行，但绝不算"高精度"。

如果把步长砍一半，\(h = 0.05\)，需要 20 步。我们不再手算（你可以拿计算器试试），结果是 \(y_{20} = 1.05^{20} \approx 2.6533\)。误差降到了 \(0.0650\)，大约一半。

这就引出了关键观察：步长砍一半，Euler 法的误差大约也砍一半。这说明 Euler 法的误差与步长成正比——也就是 21.4 节要说的"全局误差是 \(O(h)\)"。

21.3.2 一个有方向场的例子

来一个稍微正经的例子：

\[ y' = t - y^2, \qquad y(0) = 0. \]

这个方程没有闭式解。取 \(h = 0.2\)，往前算几步。\(f(t, y) = t - y^2\)。

\(y_0 = 0\) 在 \(t_0 = 0\)。\(f(0, 0) = 0\)。\(y_1 = 0 + 0.2 \cdot 0 = 0\)。在 \(t_1 = 0.2\)。

\(f(0.2, 0) = 0.2 - 0 = 0.2\)。\(y_2 = 0 + 0.2 \cdot 0.2 = 0.04\)。在 \(t_2 = 0.4\)。

\(f(0.4, 0.04) = 0.4 - 0.0016 = 0.3984\)。\(y_3 = 0.04 + 0.2 \cdot 0.3984 = 0.1197\)。在 \(t_3 = 0.6\)。

\(f(0.6, 0.1197) = 0.6 - 0.0143 = 0.5857\)。\(y_4 = 0.1197 + 0.2 \cdot 0.5857 = 0.2368\)。在 \(t_4 = 0.8\)。

\(f(0.8, 0.2368) = 0.8 - 0.0561 = 0.7439\)。\(y_5 = 0.2368 + 0.2 \cdot 0.7439 = 0.3856\)。在 \(t_5 = 1.0\)。

这样我们用 Euler 法估计了 \(y(1) \approx 0.3856\)。这就是没有闭式解时数值方法所做的全部：吐出一个数。如果想要更准的数，把 \(h\) 取小，多算几步。

21.4 Euler 法的误差

Euler 法不是免费的。每走一步，"沿切线走一段"和"沿真曲线走一段"之间是有差距的。我们要把这个差距讲清楚，因为它将决定我们到底信不信我们的数值答案。

有两种误差需要分清楚：

  • 局部截断误差（local truncation error，LTE）：如果你恰好站在真解曲线上的某一点，做一步 Euler，那么你落到的位置和真解曲线上下一时刻的位置之间的差距。
  • 全局误差（global error）：从 \(t_0\) 一路算到某个目标 \(t_N\) 时，累积下来的总误差 \(|y_N - y(t_N)|\)。

LTE 比较好分析。Taylor 展开：

\[ y(t_n + h) = y(t_n) + h y'(t_n) + \frac{h^2}{2} y''(t_n) + O(h^3). \]

Euler 法只用到了前两项 \(y(t_n) + h y'(t_n) = y(t_n) + h f(t_n, y(t_n))\)（最后一步用了 \(y' = f\)）。所以一步 Euler 漏掉的是从 \(\frac{h^2}{2} y''(t_n)\) 开始的所有高阶项：

\[ \text{LTE} = \frac{h^2}{2} y''(t_n) + O(h^3) = O(h^2). \]

注意是 \(h^2\)，不是 \(h\)：每一步的误差和 \(h^2\) 成正比。\(h\) 越小，每一步越准。

好消息说完了，说坏消息。我们要走的不止一步，要走很多步。如果你想从 \(t_0\) 算到固定的 \(T\)，需要的步数是 \(N = (T - t_0)/h\)——\(h\) 越小，步数越多。每一步都贡献一份 \(O(h^2)\) 的误差，加起来就是

\[ \text{全局误差} \sim N \cdot h^2 = \frac{T - t_0}{h} \cdot h^2 = (T - t_0) \cdot h = O(h). \]

所以全局误差是 \(O(h)\)，少一阶。这个观察就是为什么我们叫 Euler 法是"一阶方法"。

具体后果：要把全局误差砍一半，就得把 \(h\) 砍一半，意味着多算一倍的步——计算量翻倍。要把全局误差砍 100 倍，得把 \(h\) 砍 100 倍，计算量乘以 100。这种"线性"换算速度并不让人满意。如果想要 6 位精度（比如 \(10^{-6}\)），需要 \(h \sim 10^{-6}\)，从 \(t = 0\) 算到 \(t = 1\) 就要算一百万步。再去算更长的时间区间，或者解多个分量的方程组，立刻就慢得不能用。

这就是设计更高阶方法的动力：如果一个方法的全局误差是 \(O(h^p)\)，\(p\) 越大，把 \(h\) 砍一半时误差被砍的倍数就越大。RK4（21.6 节）的全局误差是 \(O(h^4)\)；步长砍一半，误差砍 \(2^4 = 16\) 倍。这种巨大的回报正是 RK4 至今仍是工业标准的原因。

顺便说一句：上面的局部 vs 全局分析，对所有"一步法"都通用。一般规律是：若局部截断误差是 \(O(h^{p+1})\)，则全局误差是 \(O(h^p)\)。这就是为什么 RK4 的局部误差需要做到 \(O(h^5)\) 才能让全局精度达到 \(O(h^4)\)。

21.5 改进 Euler 法（Heun 法）

Euler 法的"罪过"在哪里？它在整个区间 \([t_n, t_{n+1}]\) 上都假定斜率是左端点 \(t_n\) 处的斜率。但真实曲线上的斜率会变。如果右端点的斜率比左端点大，Euler 会低估真值；反之会高估。

一个朴素的改进：用区间两端斜率的平均。但这有个鸡生蛋问题——右端点的 \(y\) 是我们要算的，怎么会知道那里的斜率？

这里的窍门，是先用普通 Euler 做一个粗糙的预估，得到右端点 \(y\) 的初步值，然后用这个初步值来估算右端点的斜率，再把两个斜率取平均，重新走一步。这就是 Heun 法，有时也叫"改进的 Euler 法"或"二阶预测-校正法"。

给定 \(y' = f(t, y)\) 和当前点 \((t_n, y_n)\)：

  1. 预测：用普通 Euler 估一下右端点的 \(y\)： \[ \tilde{y}_{n+1} = y_n + h \cdot f(t_n, y_n). \]
  2. 校正：用左端点斜率和（用预测值评估的）右端点斜率的平均，重新走： \[ \boxed{y_{n+1} = y_n + \frac{h}{2}\bigl[ f(t_n, y_n) + f(t_{n+1}, \tilde{y}_{n+1}) \bigr].} \]

每一步要计算两次 \(f\)（一次在左端点，一次在用预测值评估的右端点），所以计算量是 Euler 的两倍。但精度大幅提升。

误差分析（不细做，只给结论）：Heun 法的局部截断误差是 \(O(h^3)\)，全局误差是 \(O(h^2)\)。所以它是一个二阶方法。把 \(h\) 砍一半，误差砍 4 倍——比 Euler 的"砍 2 倍"好得多。

来看一个例子。还是 \(y' = y\)，\(y(0) = 1\)，\(h = 0.1\)，目标 \(t = 1\)。

第一步。\(y_0 = 1\)，\(t_0 = 0\)，\(f(0, 1) = 1\)。预测 \(\tilde{y}_1 = 1 + 0.1 \cdot 1 = 1.1\)，斜率在 \((0.1, 1.1)\) 处是 \(1.1\)。校正 \(y_1 = 1 + 0.05 \cdot (1 + 1.1) = 1 + 0.05 \cdot 2.1 = 1.105\)。

第二步。\(y_1 = 1.105\)，\(t_1 = 0.1\)，\(f = 1.105\)。预测 \(\tilde{y}_2 = 1.105 + 0.1 \cdot 1.105 = 1.2155\)。校正 \(y_2 = 1.105 + 0.05 \cdot (1.105 + 1.2155) = 1.105 + 0.116025 = 1.221025\)。

每一步实际上是把上一步的值乘以 \(1 + 0.1 + 0.005 = 1.105\)（我们将看到，这正是 \(e^h\) 的二阶 Taylor 展开 \(1 + h + h^2/2\) 在 \(h = 0.1\) 时的值）。10 步之后：\(y_{10} = 1.105^{10} \approx 2.7141\)。

真值 \(e \approx 2.71828\)。误差 \(\approx 0.0042\)，比 Euler 法的 \(0.1245\) 小了大约 30 倍——而我们只多算了一倍的次数。这就是从一阶到二阶的回报。

21.6 Runge–Kutta 4 阶（RK4）

把上一节的"取平均斜率"思想推到极致，就得到了 1900 年前后由 Carl Runge 和 Wilhelm Kutta 发展出来的方法族。其中最著名的是经典 RK4 法——它在工程界、物理学界、化学界，几乎是默认的 ODE 求解器。下面给出完整公式，然后慢慢解释。

经典四阶 Runge–Kutta 法（RK4）： 给定 \(y' = f(t, y)\)，\(y(t_0) = y_0\)，步长 \(h\)。每一步计算四个"试探斜率"：

\[ k_1 = f(t_n,\ y_n), \] \[ k_2 = f\bigl(t_n + \tfrac{h}{2},\ y_n + \tfrac{h}{2} k_1\bigr), \] \[ k_3 = f\bigl(t_n + \tfrac{h}{2},\ y_n + \tfrac{h}{2} k_2\bigr), \] \[ k_4 = f\bigl(t_n + h,\ y_n + h k_3\bigr), \]

然后用它们的"加权平均"作为这一步的斜率：

\[ \boxed{y_{n+1} = y_n + \frac{h}{6}\bigl(k_1 + 2 k_2 + 2 k_3 + k_4\bigr),\qquad t_{n+1} = t_n + h.} \]

从公式上看吓人，从思想上看就是"在区间内取四个不同位置的斜率，按权重 \((1, 2, 2, 1)/6\) 加权平均"。这四个位置和权重不是随便选的，是为了让 Taylor 展开的前 5 阶（直到 \(h^4\)）和真解的 Taylor 展开完全匹配。结果：每一步的局部截断误差是 \(O(h^5)\)，全局误差是 \(O(h^4)\)。

我可以试着给一个直观解释。\(k_1\) 是左端点的斜率（Euler 用的就是这个）。\(k_2\) 是用 \(k_1\) 试探着走半步、在区间中点估的斜率。\(k_3\) 是用 \(k_2\) 试探着走半步（更精细一点），在中点估的另一个斜率。\(k_4\) 是用 \(k_3\) 走完整步，在右端点估的斜率。所以 \(k_1\) 是开头，\(k_4\) 是结尾，\(k_2\) 和 \(k_3\) 是两个不同的中段估计。最后取权重 \(1, 2, 2, 1\)：两端点权重小，中点权重大——这正好是 Simpson 法的权重，跟数值积分背后是同一个故事。这不是巧合：求解 ODE 在某种意义上就是积分微分方程的右端，而 Simpson 法是数值积分中四阶精度的最经济选择。

每一步要算 4 次 \(f\)（比 Heun 多一倍），但每一步精度的回报巨大。把 \(h\) 砍一半，误差砍 \(2^4 = 16\) 倍。这意味着想要 6 位精度（\(10^{-6}\)），用 RK4 大约只需要几百到几千步就够了，而 Euler 需要一百万步。RK4 真正把 ODE 的数值求解从"理论可行"变成了"实践高效"。

21.6.1 一个 RK4 例子

同样地，\(y' = y\)，\(y(0) = 1\)，\(h = 0.1\)。\(f(t, y) = y\)。

第一步。\(t_0 = 0\)，\(y_0 = 1\)。

\[ k_1 = f(0, 1) = 1. \] \[ k_2 = f(0.05,\ 1 + 0.05 \cdot 1) = f(0.05, 1.05) = 1.05. \] \[ k_3 = f(0.05,\ 1 + 0.05 \cdot 1.05) = f(0.05, 1.0525) = 1.0525. \] \[ k_4 = f(0.1,\ 1 + 0.1 \cdot 1.0525) = f(0.1, 1.10525) = 1.10525. \] \[ y_1 = 1 + \tfrac{0.1}{6}(1 + 2 \cdot 1.05 + 2 \cdot 1.0525 + 1.10525) = 1 + \tfrac{0.1}{6} \cdot 6.31025 \approx 1.10517. \]

真值 \(e^{0.1} \approx 1.10517091808\ldots\)。我们的 \(y_1 = 1.105170833\ldots\)，到小数点后 6 位都是对的——只差 \(10^{-7}\) 量级！这一步 RK4 的精度比 10 步 Euler 还要高得多。

10 步以后到 \(t = 1\)，得到 \(y_{10} \approx 2.71828169\)，真值 \(e \approx 2.71828183\)。误差是 \(\approx 1.4 \times 10^{-7}\)。比 Heun 高 5 个数量级，比 Euler 高 6 个数量级。

21.7 隐式方法初探

到目前为止，所有方法都属于显式类：右端点的值 \(y_{n+1}\) 单独出现在等号左边，由已知量算出。这类方法的优点是简单快；缺点是对所谓"刚性"方程极为不友好。

什么叫刚性？粗略地说，就是方程的解里同时存在时间尺度差异巨大的成分——既有"过几年也不变"的慢成分，也有"过 1 微秒就趋于零"的快成分。具体例子：

\[ y' = -1000(y - \cos t) - \sin t, \qquad y(0) = 1. \]

这个方程的精确解是 \(y(t) = \cos t + e^{-1000 t}\)。指数项 \(e^{-1000 t}\) 衰减得超快——\(t = 0.01\) 时已经只剩 \(e^{-10} \approx 4.5 \times 10^{-5}\)。所以从 \(t > 0.01\) 开始，解几乎就等于 \(\cos t\)，长期行为是又慢又光滑的。

问题来了：Euler 法在这个问题上稳不稳定？我们做做稳定性分析。把 Euler 用在 \(y' = \lambda y\)（线性化）上：\(y_{n+1} = y_n + h \lambda y_n = (1 + h\lambda) y_n\)。所以 \(y_n = (1 + h\lambda)^n y_0\)。这个序列衰减到 0 当且仅当 \(|1 + h\lambda| < 1\)，也就是（对 \(\lambda < 0\)）需要 \(h < 2/|\lambda|\)。

对我们的方程，本征"快尺度"是 \(\lambda \approx -1000\)，所以稳定性要求 \(h < 2/1000 = 0.002\)。即使快成分早已衰减得无关紧要，显式 Euler 仍然被那个早就死掉的快尺度钉住步长——你想用大步长 \(h = 0.1\) 跨过去？立刻数值爆炸。

救星是隐式 Euler 法（backward Euler）：

\[ \boxed{y_{n+1} = y_n + h \cdot f(t_{n+1}, y_{n+1}).} \]

注意右边的 \(f\) 是在新时刻 \(t_{n+1}\) 和新的 \(y_{n+1}\) 处求值的，所以 \(y_{n+1}\) 同时出现在等号两边——这就是"隐式"的来源。每一步要解一个方程（如果 \(f\) 是非线性的，要用 Newton 法之类）。

把它用在 \(y' = \lambda y\) 上：\(y_{n+1} = y_n + h \lambda y_{n+1}\)，所以 \(y_{n+1} = y_n / (1 - h\lambda)\)。当 \(\lambda < 0\)，\(|1/(1 - h\lambda)| < 1\) 对所有 \(h > 0\) 都成立。这意味着隐式 Euler 在 \(y' = \lambda y\)（\(\lambda < 0\)）上对任何 \(h\) 都稳定——它无条件稳定。

代价：每一步要解一个方程（多花点工夫），但你不再被快尺度绑住步长，对刚性问题大步迈进。化学反应动力学、控制系统、电路仿真，几乎一定要用隐式方法。

口头总结：

  • 非刚性问题：用显式方法（Euler / Heun / RK4），简单快。
  • 刚性问题：用隐式方法（隐式 Euler、梯形法、BDF 系列、Rosenbrock 系列），每一步贵但能跨大步。

怎么知道一个问题是不是刚性的？最简单的指示：你试着用显式方法（比如 RK4）解，如果发现"必须把 \(h\) 取得离谱地小才不爆炸，但解本身一点也不剧烈"，那就是刚性。

21.8 步长选择与误差控制

到目前为止我们一直把步长 \(h\) 当成事先选好的常数。但实际中，解曲线在某些区间变化剧烈、需要小步长才能跟上，在另一些区间又非常平缓、用大步长完全够。"全程一个固定的 \(h\)" 要么太慢（在平缓区段浪费），要么不准（在剧烈区段失真）。

解决办法是自适应步长：算法在每一步动态调整 \(h\)，使得每一步的误差控制在用户指定的容差以内。

怎么估计每一步的误差？最常用的窍门是同时跑两个不同阶数的方法。比如 RK4-RK5（也叫 Runge–Kutta–Fehlberg 4(5) 法或 RKF45）：每一步同时算一个 RK4 估值 \(y_{n+1}^{(4)}\) 和一个 RK5 估值 \(y_{n+1}^{(5)}\)。把两者之差当作误差估计：

\[ \text{误差估计} \approx \bigl| y_{n+1}^{(5)} - y_{n+1}^{(4)} \bigr|. \]

如果这个误差比用户给的容差 \(\varepsilon\) 大，说明步长过大，把 \(h\) 砍掉一半重做这一步；如果比 \(\varepsilon\) 小很多，说明步长太保守，把 \(h\) 加倍迈下一步。

更细的策略：用一个公式

\[ h_{\text{新}} = h \cdot \left( \frac{\varepsilon}{|\text{误差}|} \right)^{1/5} \cdot 0.9 \]

来直接把 \(h\) 调到目标精度（指数 \(1/5\) 来自方法的阶数加 1，\(0.9\) 是经验性的"安全系数"）。MATLAB 的 ode45、SciPy 的 solve_ivp（默认 RK45）等等，全都按这一类策略工作。这就是为什么你只要把方程交给它，它就能自动给你准确解：你不操心 \(h\)，它操心。

对于刚性问题，自适应隐式方法（如 BDF、Rosenbrock 之类）的策略类似，不过后端用的是隐式格式。MATLAB 的 ode15s、SciPy 的 'BDF' 选项就是这个家族。

建议：

  • 遇到一个新方程，先用显式自适应方法（RK45）跑跑看；
  • 如果发现它跑得很慢（步长被压得很小），说明可能是刚性，切换到隐式自适应方法；
  • 始终设置合理的容差（比如 \(10^{-6}\) 量级）；不要相信无脑用默认值。

21.10 用 Python/MATLAB 实现

真正写代码的时候，几乎没人手撸 RK4，因为现成的库已经做得很好。但写一遍能让你彻底懂怎么回事。下面给一段 Python 风格的伪码：

def euler(f, t0, y0, T, h):
    t, y = t0, y0
    history = [(t, y)]
    while t < T:
        y = y + h * f(t, y)
        t = t + h
        history.append((t, y))
    return history

def heun(f, t0, y0, T, h):
    t, y = t0, y0
    history = [(t, y)]
    while t < T:
        k1 = f(t, y)
        y_pred = y + h * k1
        k2 = f(t + h, y_pred)
        y = y + 0.5 * h * (k1 + k2)
        t = t + h
        history.append((t, y))
    return history

def rk4(f, t0, y0, T, h):
    t, y = t0, y0
    history = [(t, y)]
    while t < T:
        k1 = f(t,         y)
        k2 = f(t + h/2,   y + h/2 * k1)
        k3 = f(t + h/2,   y + h/2 * k2)
        k4 = f(t + h,     y + h   * k3)
        y = y + (h/6) * (k1 + 2*k2 + 2*k3 + k4)
        t = t + h
        history.append((t, y))
    return history

# 使用
f   = lambda t, y: -y + t
sol = rk4(f, t0=0, y0=1, T=1, h=0.25)

注意：上面的 \(y\) 既可以是标量，也可以是向量——只要 \(f\) 返回一个同形状的向量，所有公式都不变。这就是为什么数值方法天然适用于一阶方程组。如果你有一个高阶方程，比如 \(y'' + p y' + q y = g\)，先把它写成一阶方程组（令 \(z = y'\)，得到 \(y' = z\)，\(z' = g - p z - q y\)），就能用上面的代码原封不动地解。

实践中你会更多地用现成库。Python 里：

from scipy.integrate import solve_ivp

def f(t, y):
    return -y + t

sol = solve_ivp(f, [0, 1], [1], method='RK45', rtol=1e-8, atol=1e-10)
print(sol.t, sol.y)

MATLAB 里：

f = @(t, y) -y + t;
[t, y] = ode45(f, [0, 1], 1);
plot(t, y)

这两段代码做的事情和我们手撸的 RK4 一样，只是它们自动选择了步长、自动估计了误差，并且实现得比我们快得多。但你已经懂得，在这两行命令背后，发生的就是 21.6 节那四个 \(k\)、那个加权平均、那次"沿着方向场往前走一小步"。这个"魔法盒子"对你而言不再是黑箱。
