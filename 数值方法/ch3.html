第 3 章 误差怎么传：截断、舍入、向前向后

第 2 章告诉你"每次浮点 op 的相对误差不超过 \(\varepsilon_{\text{mach}}\)"。这一章问下一个问题：把成千上万次这种 op 串在一起做一段计算之后，最终答案离真答案多远？

这件事比表面看复杂。误差有两种不同来源（截断 vs 舍入），它们彼此可以互换交易；它有两种不同测法（向前 vs 向后），同一个算法在两种测法下可以一稳一不稳。这套语言是 Wilkinson 在 1950-60 年代锻造的，至今是数值分析理解算法质量的标准工具。这一章是这套语言的安装。

3.1 两种误差：截断 vs 舍入

来一道朴素任务：算 \(\sin(0.5)\)。

你大概想"用 Taylor 展开"。\(\sin(x) = x - x^3/6 + x^5/120 - x^7/5040 + \ldots\)。但你不能加无穷多项——计算机时间有限。所以你截到某一项停下来。比如截到 \(x^5/120\)：

\[ \hat S(x) = x - \frac{x^3}{6} + \frac{x^5}{120}. \]

代入 \(x = 0.5\)：\(\hat S(0.5) = 0.5 - 0.020833\ldots + 0.000260\ldots = 0.479427\ldots\)，真值 \(\sin(0.5) = 0.479425538\ldots\)。差大约 \(2 \times 10^{-6}\)。

这个 \(2 \times 10^{-6}\) 不是来自浮点——它是 Taylor 展开里被你扔掉的 \(x^7/5040\) 那一项及之后。这种"用有限项近似无穷过程"产生的误差叫

\[\boxed{\ \text{截断误差（truncation error）}: \text{把无限过程截到有限。}\ }\]

它的特征：和有限精度无关——你用纸笔算这个截断 Taylor 也会得到同样的 \(2 \times 10^{-6}\)。它是方法的误差，不是机器的误差。

现在如果你实际把 \(\hat S(0.5)\) 在 double 里算一遍，你又多了一层误差——\(0.5\) 存进浮点没问题（恰好可表达），但 \(0.5^3 = 0.125\) 也精确，除以 6 给出 \(0.020833\overline{3}\)（不可精确表达，舍入），等等。每次乘除都加一点 \(\varepsilon_{\text{mach}}\) 量级的相对误差。这叫

\[\boxed{\ \text{舍入误差（roundoff error）}: \text{有限位浮点带来的每 op 微小修改。}\ }\]

它的特征：和方法无关——你换同样的算法到 single precision，舍入误差就更大；换到任意精度库 mpmath 用 1000 位浮点，舍入误差就小到几乎为零。

两种误差的总和是你看到的总误差。但更有趣的是它们之间的互换。下一节给一个非常清晰的反差。

3.2 数值微分：两种误差互打架

给定 \(f(x)\)，要数值近似 \(f'(a)\)。最朴素的方法用差商：

\[ f'(a) \approx \frac{f(a + h) - f(a)}{h}. \]

问 \(h\) 取多小最好。直觉说"\(h\) 越小越准"。让我们看真不真。

截断误差。由 Taylor，\(f(a + h) = f(a) + h f'(a) + \frac{h^2}{2} f''(a) + O(h^3)\)。所以

\[ \frac{f(a + h) - f(a)}{h} = f'(a) + \frac{h}{2} f''(a) + O(h^2). \]

截断误差大约是 \(\frac{h}{2} |f''(a)|\)，按 \(O(h)\) 走。\(h\) 越小，截断误差越小。直觉对。

舍入误差。\(f(a + h)\) 和 \(f(a)\) 都被存为浮点，每个有 \(\sim \varepsilon_{\text{mach}} \cdot |f|\) 的绝对误差。两者相减，绝对误差不超过 \(2 \varepsilon_{\text{mach}} |f|\)。再除以 \(h\)：

\[ \text{舍入误差} \sim \frac{2 \varepsilon_{\text{mach}} |f(a)|}{h}. \]

\(h\) 越小，舍入误差越大。直觉错。

把两块加起来——总误差大约

\[ E(h) \approx \frac{|f''(a)|}{2} h + \frac{2 \varepsilon_{\text{mach}} |f(a)|}{h}. \]

这是个 V 形函数：\(h\) 太大，第一项主导；\(h\) 太小，第二项炸；中间有个最优。求导找 \(h^*\)：

\[ \boxed{\ h^* \approx \sqrt{\frac{4 \varepsilon_{\text{mach}} |f(a)|}{|f''(a)|}} \approx \sqrt{\varepsilon_{\text{mach}}} \approx 10^{-8}.\ } \]

对 double，\(h^* \approx 10^{-8}\)。比这小或比这大，都会让总误差变大。

请你停下来体会一下这件事。\(h = 10^{-8}\) 在数学上明明不是"极限的最佳近似"——数学上 \(h \to 0\) 时差商收敛到 \(f'(a)\)。但在浮点世界里，\(h\) 有个最佳尺度，下了那道线就开始倒退。"理论上越小越好"在数值上就是错的。

这条 \(h^* \approx \sqrt{\varepsilon_{\text{mach}}}\) 是数值微分的"工程公约"。你以后看科学计算里"取 step size 1e-8" 这种神秘选择——出处就在这里。也是为什么本书第 13 章专门讨论数值微分时会强调"数值微分总是比数值积分难"——积分把误差均掉，微分把误差放大。

3.3 向前误差 vs 向后误差

到现在我们说"误差"指的都是同一种东西：算出来的答案 \(\hat y\) 离真答案 \(y\) 多远。这种最朴素的测法叫

\[ \text{向前误差（forward error）} := |\hat y - y|, \quad \text{或相对版本} \frac{|\hat y - y|}{|y|}. \]

它直接、好懂、是你"想要"的那种误差。但它有个常常做不下去的痛点——"真答案" \(y\) 通常你不知道。如果你都知道真答案了，你算它干什么？

Wilkinson 1950 年代提出一个换角度的看法：与其问"我算出的答案离真答案多远"，不如问"我算出的答案是哪一道题的真答案"。

具体一点。设你在解 \(f(x) = 0\)，输入是某个函数 \(f\)，输出是某个数 \(y = 0\) 的根。算法给你 \(\hat y\)。向前误差问：\(|\hat y - y|\) 多大？向后误差问：是否存在一个被微微扰动的函数 \(\tilde f\)，使 \(\hat y\) 是它的真根？如果存在，扰动 \(\|\tilde f - f\|\) 多小？

形式化：

\[ \boxed{\ \text{向后误差（backward error）} := \min\{\|\Delta\| : \hat y = \text{exact answer to perturbed input } x + \Delta\}.\ } \]

这一句话听起来像换汤不换药。但它实际上极有威力。原因是：

  • 你能算向后误差——只需把 \(\hat y\) 代回原方程看残差，不用知道真答案。
  • 向后误差小，意味着你的算法"给出了某道附近问题的精确答案"。这是"算法尽职"的一种很自然的标准——你不能要求算法把无解决精度的问题解出来，但你可以要求算法不引入额外误差。
  • 向前误差 = 向后误差 × 问题的某种"敏感度"。这个"敏感度"叫条件数，是下一章的主角。把这两块解耦让人能分开讨论"问题难"和"算法烂"。

用数值线性代数里最熟的例子说一下。要解 \(A x = b\)，算法给 \(\hat x\)。

  • 向前误差：\(\|\hat x - x_{\text{true}}\|\)。
  • 向后误差：\(\|A \hat x - b\| = \|r\|\)，称残差。

残差告诉你"\(\hat x\) 是不是某个右端略不同的方程组的精确解"——是的，它是 \(A \hat x = b - r\) 的精确解，扰动量是 \(\|r\|\)。残差小 ⟺ 向后误差小。

关键事实（第 16 章会证）：高斯消元 + 部分主元的 LU 分解向后稳定，意思是产出的 \(\hat x\) 满足 \(\|r\| / \|b\| \lesssim n \varepsilon_{\text{mach}}\)，无论 \(A\) 多病态。所以高斯消元"把它能做的都做对了"——剩下的不准是问题本身的过错。这就是为什么 LU 即使在 \(A\) 几乎奇异时也是数值线性代数的工业标准。

3.4 误差怎么累积：不是简单求和

朴素的想法：做 \(n\) 次浮点 op，每次 \(\varepsilon_{\text{mach}}\) 误差，累积起来 \(n \cdot \varepsilon_{\text{mach}}\)。常见但不准确。

这种线性累积只在"误差永远同号"时成立。实际中浮点舍入是"四舍五入"——舍上和舍下大致一半一半，舍入误差像随机变量。如果误差独立同分布、零均值，累积之后方差线性增长，标准差按 \(\sqrt{n}\) 走。这是为什么实践中长求和的舍入误差经常是 \(\sqrt{n} \cdot \varepsilon_{\text{mach}}\)，比 \(n \cdot \varepsilon_{\text{mach}}\) 好得多。

另一极端：误差不独立、被算法系统性放大——这是灾难性抵消的来源。比如算 \(\sum_{i=1}^n x_i\) 用从大到小排序的输入，反复地"大数 + 小数 = 大数"，小数被吃掉。这种误差不是随机的，是结构性的。

三个层级：

  • 最坏情况上界：\(n \cdot \varepsilon_{\text{mach}}\)。永远成立但常常很悲观。
  • 统计典型情况：\(\sqrt{n} \cdot \varepsilon_{\text{mach}}\)。如果误差近似独立、零均值。
  • 结构性放大：可以远超 \(n \cdot \varepsilon_{\text{mach}}\)，比如灾难性抵消让单 op 的误差膨胀几个数量级。

分析一个具体算法时你要判断它属于哪一类。判断方法不是套公式，是逐 op 追踪——这就是误差分析本身（多年来发展成专门技术，Higham 那本《Accuracy and Stability of Numerical Algorithms》是这门艺术的圣经）。

本书后面遇到的算法，我会告诉你它的误差等级；但我也希望你养成自己看一眼算法、就能猜出它属于哪一类的肌肉。线性累积是默认猜测；如果你看到"长链同号加" 或 "几乎相等的数相减"，把猜测调高。

3.5 一个非常实用的求和算法：Kahan 求和

给一个具体例子。算 \(\sum_{i=1}^{10^7} x_i\)，所有 \(x_i\) 都是 1。真答案 \(10^7\)。

朴素算法（从左到右加）：每步 \(s \leftarrow s + x_i\)。当 \(s\) 足够大（比如到 \(10^6\)），\(s + 1\) 在 double 下还是精确的（因为 \(10^6\) 的相对精度还有 \(10^{-10}\) 余裕）；但如果 \(x_i\) 是更小的数（比如 \(10^{-10}\)），\(s + x_i\) 在 \(s = 10^6\) 时就开始丢精度。

这种"小被大吃" 是浮点求和的根本问题。Kahan 求和的小聪明：用一个"补偿变量" \(c\) 记录上一步丢失的低位。

sum = 0
c = 0           # 补偿（compensation）
for x in xs:
    y = x - c           # 把上一步丢的低位"还给" x
    t = sum + y         # 这里 t 的低位还会丢
    c = (t - sum) - y   # 那部分丢的低位 = 这次没加进去的
    sum = t

这个算法的正确性是个简洁的代数验证（写出每步浮点近似 \(\text{fl}(\cdot) = (\cdot)(1 + \delta_i)\)，把所有 \(\delta\) 项追踪一遍）。结论：Kahan 求和的误差不依赖 \(n\)，只依赖输入的最大幅值，是 \(O(\varepsilon_{\text{mach}})\) 而不是 \(O(n \varepsilon_{\text{mach}})\) 或 \(O(\sqrt n \varepsilon_{\text{mach}})\)。

你可能想"这是个魔术"。其实不是——它就是把"每次浮点加法丢掉的低位" 用一个变量记下来，下一次加进去。直觉上像在记账：不平账，加一笔补正。这种"补偿" 思路在浮点算法里反复出现（Dekker、Veltkamp、TwoSum 等同一族算法都用类似技巧）。

Kahan 求和的代价是 4 倍 op，但误差量级低几个数量级。你做大量小数求和（统计、机器学习、信号处理）时几乎总应该用它。

3.6 一个让你以后受用的小工具：相对 vs 绝对误差

读完这一章，你应该养成一个反射：谈误差时先问"这是相对的还是绝对的"。

  • 相对误差：\(|\hat y - y| / |y|\)。无量纲，跨问题可比。
  • 绝对误差：\(|\hat y - y|\)。有量纲，跟 \(y\) 的尺度走。

浮点的"\(\varepsilon_{\text{mach}} = 2 \times 10^{-16}\)" 是相对精度。double 算 \(10^{20}\) 量级的数，绝对精度只有 \(10^4\)；算 \(10^{-20}\) 量级的数，绝对精度 \(10^{-36}\)。同样的相对精度，绝对精度按数量级伸缩。这就是第 2 章 2.4 节反复说的"相对精度均匀，绝对精度伸缩"。

例外：当真值接近 0 时，相对误差爆掉，这时绝对误差更有意义。比如算 \(\sin(\pi)\)（真值 0），双精度结果大约 \(1.2 \times 10^{-16}\)。绝对误差 \(1.2 \times 10^{-16}\) 是合理的"接近 0"；相对误差却是 100%（除以 0 的极限）。所以"相对误差"在真值附近 0 时不是好的衡量。

实际工作中：用混合标准，例如

\[ |\hat y - y| < \tau_{\text{abs}} + \tau_{\text{rel}} \cdot |y|. \]

当 \(|y|\) 小，\(\tau_{\text{abs}}\) 主导；当 \(|y|\) 大，\(\tau_{\text{rel}} \cdot |y|\) 主导。这是 SciPy / MATLAB 等库的默认收敛判据格式。

下一章我们正式定义条件数——一个不依赖算法、只依赖问题本身的"先天难度"。条件数大的问题，再好的算法也救不回来；条件数小的问题，再烂的算法也错不到哪去。理解了条件数你就理解了"算法没问题，是问题本身病" 这种困境。
