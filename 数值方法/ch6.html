第 6 章 一维方程求根：二分与不动点

前五章是地基——浮点、误差、条件数、稳定性。从这一章开始我们做具体事。第一件具体事：解一个方程

\[ f(x) = 0. \]

这是数值方法最朴素也最常见的任务。物理里求平衡点、化学里算化学势、金融里反算隐含波动率、机器学习里求最大似然估计的零导数点——本质上都是解 \(f(x) = 0\)。\(f\) 通常没有闭式根（除了线性、二次、少数三次和一些三角方程），剩下的全得用数值方法。

这一章给最朴素的两个算法：二分法（永远收敛但慢）和不动点迭代（可能很快也可能发散）。下一章给把这两条优点合一的牛顿法。

6.1 为什么需要数值求根

你心想"二次方程不是有公式吗？"对，二次有。三次也有（Cardano 公式），但已经丑得没人记。四次也有（Ferrari），更丑。五次以上没有用根式表达的一般公式（这是 Galois 与 Abel 的著名结论，详见《抽象代数读本》第 22 章）。

但这只是冰山一角。物理与工程里大部分方程根本不是多项式：

  • 普朗克黑体辐射峰值：\(5(1 - e^{-x}) = x\)，没有闭式解。
  • Kepler 方程（行星运动）：\(M = E - e \sin E\)，求 \(E\)。
  • 反算 Black–Scholes 隐含波动率：\(\sigma\) 在 erfc 复合里。

这些都是"有解，但解写不出来" 的问题。数值方法不是 plan B——它是唯一的解法。

整章我们用同一个简单例子练手：

\[ f(x) = x^3 - 2 = 0, \]

真根 \(\sqrt[3]{2} \approx 1.2599210\). 这道题能闭式解（\(x = 2^{1/3}\)），但我们假装不能——这样可以把数值算出的每一步和真值对照。

6.2 二分法

二分法的全部逻辑就一句话：如果一个连续函数在区间两端取异号值，那它在中间某处必为零。这是中值定理（IVT）。给定 \(a < b\) 满足 \(f(a) \cdot f(b) < 0\)，根一定在 \([a, b]\) 内。

算法：取中点 \(m = (a + b) / 2\)，算 \(f(m)\)。三种情况：

  • \(f(m) = 0\)：找到根，停。
  • \(f(m) \cdot f(a) < 0\)：根在 \([a, m]\)，把 \(b\) 缩到 \(m\)。
  • 否则根在 \([m, b]\)，把 \(a\) 缩到 \(m\)。

循环。每步把区间长度减半。

对 \(f(x) = x^3 - 2\) 起点 \([1, 2]\)，\(f(1) = -1, f(2) = 6\)：

每步区间减半。从 \([1, 2]\) 长度 1 出发，要到 \(\varepsilon_{\text{mach}} = 2^{-52}\)，恰好需要 52 步。这是二分法的"线性收敛"——每步把误差减半，等价于"每步多一位二进制精度"。

\[ \boxed{\ \text{二分法误差} \le \frac{b - a}{2^n}, \quad \text{每步精度增加 1 位.}\ } \]

53 步算 16 位精度——比起牛顿法的"每步精度翻倍"（下一章），慢得多。但它的承诺也强得多：给定有效起始括号 \([a, b]\)，二分法 100% 收敛。这种"绝不失败" 的稳定性在工程上极有价值。

6.3 二分法的细节陷阱

看似简单的二分法在实现上有几个让你以后受用的细节。

陷阱 1：中点的算法。朴素 \(m = (a + b) / 2\) 在 \(a, b\) 数量级很大时可能上溢。比如 \(a = 10^{300}\)，\(b = 1.5 \times 10^{300}\)，加起来 \(2.5 \times 10^{300}\)没溢出，但如果 \(b = 10^{308}\) 之类就接近 double 上限了。更稳的写法：

\[ m = a + (b - a) / 2. \]

\(b - a\) 是个相对小的数，加 \(a\) 不会溢出。这个小重排是无代价的好习惯。

陷阱 2：停止判据。什么时候认定"找到根了"？不是"区间长度小" 单一标准——病态问题里 \(f(m)\) 仍可能很大。也不是"\(|f(m)|\) 小" 单一标准——\(f\) 可能本来就接近 0（比如 \(f(x) = (x - 1)^{10}\) 在 \(x = 1\) 附近的所有点 \(f\) 都很小，但根的位置仍要精确）。

实战的双判据：

\[ |b - a| < \tau_x \quad \textbf{且} \quad |f(m)| < \tau_f. \]

\(\tau_x\) 通常取 \(\varepsilon_{\text{mach}} \cdot |x|\)（相对位置精度），\(\tau_f\) 取问题特定的小数。两个都满足才停。

陷阱 3：起始括号的获取。二分法需要一个 \(f(a) \cdot f(b) < 0\) 的括号——但怎么找这个括号本身是另一个问题。常见方法：

  • 从问题的物理 / 几何意义猜（一个温度问题，温度大概在 \([0, 100]\) 摄氏度内）。
  • 采样：\(f\) 在 \([x_{\min}, x_{\max}]\) 上每隔 \(\Delta x\) 取一点，找连续两个符号异的点。
  • "扩张" 算法：从 \(x_0\) 出发，每次把区间扩大一倍 \([x_0 - 2^k, x_0 + 2^k]\)，直到符号异。

没找到括号？说明 \([a, b]\) 内可能无实根、或偶数个根。二分法不适用。

陷阱 4：多根。二分法找到的是某一个根，不是所有根。如果 \(f\) 在你的范围内有多个根，二分法找哪个取决于初始括号怎么选。要找全部根，你得做"扫描 + 局部二分"。

6.4 不动点迭代：另一种视角

从另一个角度看求根。把 \(f(x) = 0\) 重写成

\[ x = g(x) \]

对某个函数 \(g\)。\(g\) 不止一种选——比如 \(x^3 - 2 = 0\) 可写成 \(x = x - (x^3 - 2)\) 或 \(x = (x + 2/x^2)/2\) 或 \(x = (2/x)^{1/2}\) 等许多形式，每个都对。

解 \(x = g(x)\) 的方程叫求 \(g\) 的不动点（fixed point）。算法：从某个 \(x_0\) 出发，迭代

\[ x_{n+1} = g(x_n), \quad n = 0, 1, 2, \ldots \]

如果序列收敛到 \(x^*\)，则 \(x^* = g(x^*)\)，找到根。

例：解 \(x = \cos(x)\)。从 \(x_0 = 1\) 出发：

6.5 不动点迭代什么时候收敛？

核心定理（压缩映射 / Banach 不动点定理的实数版）：

定理。设 \(g\) 在 \(x^*\) 附近连续可微，\(x^* = g(x^*)\) 是不动点。如果 \(|g'(x^*)| < 1\)，则存在邻域 \(U(x^*)\) 使任何 \(x_0 \in U\) 出发的迭代收敛到 \(x^*\)。收敛速率：

\[ \boxed{\ |x_{n+1} - x^*| \approx |g'(x^*)| \cdot |x_n - x^*|.\ } \]

每步误差被 \(|g'(x^*)|\) 倍缩。叫线性收敛，速率 \(|g'(x^*)|\)。

反过来，如果 \(|g'(x^*)| > 1\)，迭代发散——任何起点最终都跑离 \(x^*\)。\(|g'(x^*)| = 1\) 是临界，可能收敛也可能不，看高阶项。

验证两个例子：

\(g(x) = \cos(x)\)，\(x^* \approx 0.739\)。\(g'(x^*) = -\sin(0.739) \approx -0.673\)。\(|g'(x^*)| \approx 0.673 < 1\) → 收敛，每步误差缩 0.67 倍。这就是上一节我们看到的"慢慢震荡收敛"。

\(g(x) = 2/x\)，\(x^* = \sqrt{2}\)。\(g'(x) = -2/x^2\)。\(g'(\sqrt{2}) = -2/2 = -1\)。\(|g'(x^*)| = 1\) → 临界，本例中实际不收敛（永远震荡）。

\(g'(x^*)\) 越接近 0，收敛越快。理想是 \(g'(x^*) = 0\)——这时收敛是二次的（每步精度翻倍），比线性快得多。怎么构造让 \(g'(x^*) = 0\) 的 \(g\)？这就是下一章牛顿法的精髓——它本质就是个"自动选最优 \(g\)" 的算法。

请你停下来想一件事：上面这条导数大小决定收敛与否 的事实，把"求根" 和"几何" 联系起来。\(g'(x^*) < 0\) 让迭代震荡（每步换边）；\(g'(x^*) > 0\) 让迭代单调；\(|g'(x^*)|\) 大让收敛慢甚至发散。整个不动点理论是 \(g\) 在不动点附近的局部线性近似的几何。这种"用线性近似理解非线性算法" 的视角在数值方法里反复出现。

6.6 二分 vs 不动点：怎么选

这两种方法在工程上的取舍：

二分法不动点迭代
需要异号括号 \(f(a) f(b) < 0\)良好初值 + 合适 \(g\)
收敛保证100%（连续 + 异号）仅 \(|g'| < 1\) 时
收敛速度每步 1 位（线性）线性，速率 \(|g'(x^*)|\)
每步开销1 次 \(f\) 求值1 次 \(g\) 求值
易实现极易易但需想 \(g\)
病例无括号、多根\(g\) 选错则发散

实战做法：用组合算法。Brent's method 是工业标准——它开始用二分保证收敛进入根附近，切换到二次插值或反向二次插值加速到接近 Newton 速度，但每步监控收敛性，不行就退回二分。这种"多算法组合" 的思路在数值方法里反复出现：每个单算法各有优劣，组合起来取长补短。

SciPy 的 scipy.optimize.brentq 就是 Brent 算法的实现。99% 的"找一维方程根" 任务直接用它就行。理解二分和不动点是为了让你知道它底层在做什么、什么时候会失败。

下一章我们引入牛顿法。它本质是"用 \(f\) 在 \(x_n\) 处的切线代替 \(f\) 找新猜值" 的不动点迭代——但选 \(g\) 选得极好，使得 \(g'(x^*) = 0\)，得到二次收敛（每步精度翻倍）。代价是每步要算 \(f\) 和 \(f'\)，且不像二分那样保证收敛。下一章我们看清楚这个交易。
