第 8 章 多变量牛顿与拟牛顿

一维牛顿法用切线代替函数。多维牛顿法把"切线" 换成"切超平面"——本质是多元 Taylor 一阶展开。整个故事的逻辑和一维一样，但规模带来新问题：Jacobian 是 \(n \times n\) 矩阵，每步要解一次线性方程组（开销 \(O(n^3)\)），\(n\) 大时这两条都要命。

这一章先正面给多维牛顿法。然后引入拟牛顿系列——通过近似Jacobian 把每步从 \(O(n^3)\) 降到 \(O(n^2)\) 甚至 \(O(n)\)。后者是机器学习里 BFGS / L-BFGS 等优化算法的源头。

8.1 多维求根问题的形状

设 \(F : \mathbb{R}^n \to \mathbb{R}^n\)，要解 \(F(x) = 0\)——即 \(n\) 个方程 \(n\) 个未知数。展开来看：

\[ \begin{cases} f_1(x_1, \ldots, x_n) = 0 \\ f_2(x_1, \ldots, x_n) = 0 \\ \vdots \\ f_n(x_1, \ldots, x_n) = 0 \end{cases} \]

具体例子：

  • 电路分析：基尔霍夫电流定律 + 欧姆定律给一组非线性方程，未知数是各节点电压。
  • 力学平衡：合力 \(\sum F_i = 0\) 是矢量方程，每个分量给一个标量方程。
  • 化学平衡：质量作用律给一组多项式方程，未知数是各物种浓度。
  • 反算 ODE 隐式格式：\(x_{n+1} = x_n + h \cdot f(x_{n+1})\) 在每个时间步都是一道非线性方程组（第 23 章）。

多维问题比一维难有几个原因。

第一，解的几何更复杂。一维 \(f(x) = 0\) 的解集通常是孤立点。多维 \(F(x) = 0\) 的解集可以是点、线、面、流形——取决于 \(F\) 雅可比的秩。如果 Jacobian 的秩满（\(n\)），解是孤立的；如果不满，解是更高维的子流形（系统欠定）。

第二，没有"括号"概念。一维有 IVT 让你可靠"夹住" 一个根；多维没有自然推广。所以二分法在多维不适用，必须靠迭代法。

第三，多个根难处理。一维你可以扫描区间找全部异号点；多维空间没法"扫描" \(\mathbb{R}^n\)。一般你只能用多个不同初值各跑一次牛顿，希望覆盖到不同根。这是一种"蒙特卡洛式" 求全部根。

8.2 多维牛顿法

把 \(F\) 在当前 \(x_n\) 处线性化：

\[ F(x) \approx F(x_n) + J(x_n)(x - x_n), \]

其中 \(J(x_n)\) 是 \(n \times n\) 的 Jacobian 矩阵，\(J_{ij} = \partial f_i / \partial x_j\)。

令右边等于 0 解 \(x\)：

\[ J(x_n)(x - x_n) = -F(x_n) \quad \implies \quad x = x_n - J(x_n)^{-1} F(x_n). \]

所以多维牛顿迭代：

\[ \boxed{\ x_{n+1} = x_n + \Delta x_n, \quad J(x_n) \Delta x_n = -F(x_n).\ } \]

注意这里写成 \(J \Delta x = -F\) 而不是 \(\Delta x = -J^{-1} F\)。原因是：你从不显式求逆矩阵——求逆慢且不稳（第 16 章会专门讨论）。你解线性方程组，开销 \(O(n^3)\) 用 LU 分解，比求逆便宜也更稳。这条小语义差别在数值线性代数里很重要："\(J^{-1} F\)" 是数学符号，"解 \(J \Delta x = -F\)" 是算法。

8.3 跑一遍 2D 例子

解

\[ \begin{cases} f_1 = x^2 + y^2 - 4 = 0 & \text{(圆 } r = 2\text{)}\\ f_2 = x y - 1 = 0 & \text{(双曲线 } x y = 1\text{)} \end{cases} \]

几何上：两条曲线相交。圆与 \(xy = 1\) 双曲线相交于第一象限两点，第三象限两点。第一象限两点近似 \((1.93, 0.52)\) 和 \((0.52, 1.93)\)。

Jacobian：

\[ J = \begin{pmatrix} \partial f_1 / \partial x & \partial f_1 / \partial y \\ \partial f_2 / \partial x & \partial f_2 / \partial y \end{pmatrix} = \begin{pmatrix} 2x & 2y \\ y & x \end{pmatrix}. \]

从 \((2, 0.5)\) 出发：

步 1：\(F = (4 + 0.25 - 4, 1 - 1) = (0.25, 0)\)。\(J = \begin{pmatrix} 4 & 1 \\ 0.5 & 2 \end{pmatrix}\)。解 \(J \Delta = -(0.25, 0)\) 给 \(\Delta = (-0.0667, 0.0167)\)。新 \(x = (1.933, 0.517)\)。

步 2：\(F \approx (0.0011, 0.0001)\)，已经几乎为零。再做一步把误差压到 \(\sim 10^{-8}\)。再一步到 \(\sim 10^{-15}\)。

三四步收敛到机器精度——和一维牛顿同等的速度。这就是"每步精度翻倍" 在多维的体现。

8.4 局部二次收敛（多维版）

定理。设 \(F \in C^2\)，\(F(x^*) = 0\)，\(J(x^*)\) 非奇异。则存在 \(x^*\) 邻域内任意起点的牛顿迭代收敛到 \(x^*\)，且

\[ \|x_{n+1} - x^*\| \le C \|x_n - x^*\|^2. \]

证明思路同一维（多元 Taylor 加 \(J(x^*)^{-1}\) 范数估计）。详见 Nocedal–Wright《Numerical Optimization》第 11 章。

关键前提换成"\(J(x^*)\) 非奇异"——即 Jacobian 行列式非零。这个条件在解的几何是孤立点时通常满足。如果 \(J(x^*)\) 奇异，意味着解附近 \(F\) 的线性化退化——通常对应解的"高重数" 或"沿某方向连续解"。这种退化情形需要专门处理（如修正牛顿、半光滑方法等），本书不深入。

失效模式和一维类似但更微妙：

  • \(J(x_n)\) 几乎奇异 → \(\Delta x\) 很大 → 跳得太远 → 发散。修法：步长 \(\alpha < 1\)（line search，下面 8.7 节）。
  • 初值远 → 不在收敛域 → 跳到错误根或发散。修法：好的初值、全局策略（trust region、homotopy 等）。
  • 解集是连续流形 → \(J\) 在整个流形上奇异 → 牛顿法无法收敛到孤立点。修法：用 SVD-based 伪逆，或者重新参数化。

8.5 Jacobian 太贵：拟牛顿法

多维牛顿法的实际瓶颈是计算 Jacobian。\(n \times n\) 矩阵有 \(n^2\) 个偏导。每个偏导可能要：

  • 符号求导（复杂表达式时不实际）；
  • 自动微分（现代框架如 JAX / PyTorch 支持，但每步开销仍大）；
  • 有限差分 \(\partial f_i / \partial x_j \approx (f_i(x + h e_j) - f_i(x))/h\)，需要 \(n+1\) 次 \(F\) 求值。

对 \(n = 1000\)，每步 \(\sim 10^6\) 次标量函数求值（如果 \(F\) 本身复杂），完全不可接受。

拟牛顿法（quasi-Newton）的核心思想：不算精确 Jacobian，用上一步的（\(\Delta x, \Delta F\)）信息更新一个 Jacobian 近似 \(B_n \approx J(x_n)\)。每步开销从 \(O(n^2 \cdot \text{cost of } f) + O(n^3)\) 降到 \(O(n^2)\) 或 \(O(n)\)（带稀疏结构）。

最简单的Broyden 法（"多维割线法"）：用 rank-1 更新维护 \(B_n\)：

\[ B_{n+1} = B_n + \frac{(\Delta F - B_n \Delta x) \Delta x^T}{\Delta x^T \Delta x}, \quad \Delta x = x_{n+1} - x_n, \quad \Delta F = F(x_{n+1}) - F(x_n). \]

这个更新让 \(B_{n+1} \Delta x = \Delta F\)（"割线条件"——多维割线类比）。其它方向上 \(B_{n+1}\) 和 \(B_n\) 一致。

Broyden 法收敛超线性（介于线性和二次之间），但比真牛顿便宜得多。在求解非线性方程组时是个常用工具。

8.6 优化里的拟牛顿：BFGS 与 L-BFGS

多维牛顿最常见的应用不是解 \(F(x) = 0\)，是最小化 一个标量函数 \(g(x)\)：

\[ \min_{x \in \mathbb{R}^n} \ g(x). \]

极小点处 \(\nabla g(x^*) = 0\)，所以问题归约为求根 \(F(x) = \nabla g(x) = 0\)。Jacobian \(J = \nabla^2 g = H\)（Hessian 矩阵，对称的）。

牛顿法的迭代：

\[ x_{n+1} = x_n - H(x_n)^{-1} \nabla g(x_n). \]

问题同前——Hessian 贵。BFGS（Broyden–Fletcher–Goldfarb–Shanno，四个发明人，1970）是优化里最有名的拟牛顿，rank-2 更新近似 Hessian 的逆：

\[ B_{n+1}^{-1} = (I - \rho s y^T) B_n^{-1} (I - \rho y s^T) + \rho s s^T, \]

其中 \(s = \Delta x\), \(y = \Delta(\nabla g)\), \(\rho = 1/(y^T s)\)。这套更新保证：(1) 满足割线条件；(2) 保持对称性；(3) 保持正定（收敛邻域内）。

BFGS 的好处：每步 \(O(n^2)\)，超线性收敛，常常实战表现接近真牛顿。但需要存 \(n \times n\) 的 \(B^{-1}\)——\(n = 10^6\) 时存不下。

L-BFGS（Limited-memory BFGS）解决这个：只存最近 \(m\) 个 \((s_k, y_k)\) 对（典型 \(m = 5\) 到 20），每步用它们隐式应用 BFGS 更新。每步开销 \(O(m n)\)，存储 \(O(m n)\)。这就是大规模优化的工业标准——SciPy 的 L-BFGS-B、TensorFlow 的二阶优化器、几乎所有 logistic regression / 线性 SVM 训练的底层求解器，都是 L-BFGS。

对比一阶方法（梯度下降 \(x_{n+1} = x_n - \alpha \nabla g\)）：梯度下降线性收敛，速率取决于 \(g\) 的条件数；BFGS / L-BFGS 超线性收敛。在中等规模光滑优化（10K–100M 维）上 L-BFGS 通常胜出。机器学习里训练神经网络则常用一阶方法（SGD + momentum、Adam），原因是数据集太大、随机近似梯度更适合，而不是 L-BFGS 不好。

8.7 全局收敛策略：line search 与 trust region

纯牛顿法局部收敛——初值要够近。要把它做成全局收敛（任何起点都收敛到某个局部解），常用两套策略：

Line search（线搜索）。牛顿法每步给一个方向 \(d_n = -J^{-1} F\) 或 \(-H^{-1} \nabla g\)。这是方向，不一定是合适步长。Line search 在这个方向上找最优步长 \(\alpha\)：

\[ x_{n+1} = x_n + \alpha_n d_n, \quad \alpha_n = \arg\min_\alpha g(x_n + \alpha d_n). \]

实战中不真的求精确最小，用Wolfe 条件做"足够下降" 检查。这套方法保证 \(g(x_{n+1}) < g(x_n)\)——单调下降——所以全局收敛到某个驻点。

Trust region（置信域）。不沿固定方向走，而是在 \(x_n\) 附近某个球 \(\|x - x_n\| \le r_n\) 内最小化局部二次模型 \(m_n(x) = g(x_n) + \nabla g^T (x - x_n) + \frac{1}{2}(x - x_n)^T H_n (x - x_n)\)。如果新点真的让 \(g\) 下降足够多，扩大 \(r_n\)；否则缩小。

两种思路各有优劣。Line search 实现简单，trust region 在 \(H\) 不正定（出现负曲率方向）时更稳健。SciPy 的 trust-ncg、trust-krylov 是 trust region 实现；BFGS、L-BFGS-B 是 line search 实现。

这两套策略让"局部好快" 的牛顿法变成"全局可靠 + 局部好快" 的工程级算法。所有现代非凸优化器都是这种"方向 by quasi-Newton + 步长 by line search/trust region" 的混合架构。
