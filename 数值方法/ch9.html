第 9 章 多项式插值：Lagrange 与 Newton 差商

这一章换主题：从"求根"到"插值"。设你有 \(n + 1\) 个数据点 \((x_0, y_0), \ldots, (x_n, y_n)\)，想找一个函数 \(p\) 满足 \(p(x_i) = y_i\) 对所有 \(i\)。这种问题在工程里出现得比求根还频繁——你做实验测了 10 个温度下的电阻，想画"电阻 vs 温度"的曲线；你做模拟得了 100 个点的数据，想算它的积分；你训练机器学习模型，本质就是用一堆点拟合一个函数。

这一章我们用多项式做插值——\(p\) 是次数 \(\le n\) 的多项式。这是最朴素的选择，也是数值分析最早的研究对象之一。我们会看到它在低维（\(n \le 10\)）下表现良好，并在下一章看到它在高维下灾难性失败（Runge 现象），那时再找补救（样条，第 11 章）。

9.1 插值问题

给定 \(n + 1\) 对数据点 \((x_0, y_0), \ldots, (x_n, y_n)\)，节点 \(x_0 < x_1 < \cdots < x_n\) 互异。要找插值多项式

\[ p(x) = a_0 + a_1 x + a_2 x^2 + \cdots + a_n x^n \]

满足 \(p(x_i) = y_i\) 对所有 \(i = 0, 1, \ldots, n\)。

事实 1：唯一性。这样的多项式存在且唯一。

证明：把 \(p(x_i) = y_i\) 写开是 \(n + 1\) 个线性方程，未知数是 \(a_0, \ldots, a_n\)（\(n + 1\) 个），系数矩阵是 Vandermonde 矩阵

\[ V = \begin{pmatrix} 1 & x_0 & x_0^2 & \cdots & x_0^n \\ 1 & x_1 & x_1^2 & \cdots & x_1^n \\ \vdots & & & & \vdots \\ 1 & x_n & x_n^2 & \cdots & x_n^n \end{pmatrix}. \]

它的行列式 \(\det V = \prod_{i < j}(x_j - x_i) \neq 0\) 因 \(x_i\) 互异。所以方程组有唯一解。

事实 2：用 Vandermonde 求解很糟。虽然存在唯一解，但求它的方式很多。最朴素的"组装 \(V\) 然后解线性方程组" 在数值上很烂——\(V\) 的条件数随 \(n\) 指数爆炸（\(n = 20\) 时 \(\kappa(V) \sim 10^{15}\)），求出的 \(a_i\) 完全不准。

所以需要更稳定的算法构造同一个多项式。下面两节给两种主流方法：Lagrange 形式（直接、漂亮）和 Newton 差商形式（增量、高效）。两者最终都给同一个 \(p\)（唯一性保证），但表达和计算方式不同。

9.2 Lagrange 形式

构造一组特殊的"基础多项式" \(L_0, L_1, \ldots, L_n\)：

\[ L_i(x) = \prod_{j \neq i} \frac{x - x_j}{x_i - x_j}. \]

每个 \(L_i\) 是 \(n\) 次多项式。关键性质：

\[ L_i(x_j) = \begin{cases} 1 & \text{若 } j = i \\ 0 & \text{若 } j \neq i \end{cases} = \delta_{ij}. \]

验证：\(L_i(x_i) = \prod_{j \neq i} (x_i - x_j)/(x_i - x_j) = 1\)；\(L_i(x_k)\) 当 \(k \neq i\) 时分子有 \((x_k - x_k) = 0\) 一项。

插值多项式

\[ \boxed{\ p(x) = \sum_{i=0}^n y_i L_i(x).\ } \]

验证：\(p(x_j) = \sum_i y_i L_i(x_j) = \sum_i y_i \delta_{ij} = y_j\)。

例：3 个点 \((0, 1), (1, 3), (2, 2)\)。

\(L_0(x) = (x - 1)(x - 2) / ((0 - 1)(0 - 2)) = (x-1)(x-2)/2\)。
\(L_1(x) = x(x - 2) / ((1)(- 1)) = -x(x-2)\)。
\(L_2(x) = x(x - 1) / ((2)(1)) = x(x-1)/2\)。

\(p(x) = 1 \cdot (x-1)(x-2)/2 + 3 \cdot (-x(x-2)) + 2 \cdot x(x-1)/2\)。

展开：\(p(x) = -3x^2/2 + 7x/2 + 1\)。验证：\(p(0) = 1\)，\(p(1) = -3/2 + 7/2 + 1 = 3\)，\(p(2) = -6 + 7 + 1 = 2\)。三个都对。

Lagrange 的好处：完全显式，没有线性方程组要解。坏处：

  • 每次求值 \(p(x)\) 都要算 \(n + 1\) 个 \(L_i(x)\)，每个 \(L_i\) 是 \(n\) 个因子的乘积——共 \(O(n^2)\) 操作。如果你要在很多 \(x\) 上求 \(p\)，这慢。
  • 加一个新点（\(n \to n + 1\)）后所有 \(L_i\) 都要重算——所有的"\(\prod\)" 多了一个因子。
  • 数值稳定性中等——精度可以接受，但不算最好。

这三条让 Lagrange 在理论分析里好用（公式干净），但在实战工程里很少直接用。下一节的 Newton 差商更实用。

9.3 Newton 差商形式

同一个 \(p\) 换一种写法。增量构造：

\[ p(x) = c_0 + c_1 (x - x_0) + c_2 (x - x_0)(x - x_1) + \cdots + c_n \prod_{j=0}^{n-1}(x - x_j). \]

\(c_i\) 是要求的系数。条件 \(p(x_i) = y_i\) 给一个三角线性方程组（每加一个 \(x_i\)，前面所有项在它处的值都有计算公式），逐步解出 \(c_i\)。

系数有显式公式——叫差商（divided differences）：

\[ \begin{aligned} f[x_i] &= y_i, \\ f[x_i, x_{i+1}] &= \frac{f[x_{i+1}] - f[x_i]}{x_{i+1} - x_i}, \\ f[x_i, x_{i+1}, x_{i+2}] &= \frac{f[x_{i+1}, x_{i+2}] - f[x_i, x_{i+1}]}{x_{i+2} - x_i}, \\ &\vdots \\ f[x_0, x_1, \ldots, x_k] &= \frac{f[x_1, \ldots, x_k] - f[x_0, \ldots, x_{k-1}]}{x_k - x_0}. \end{aligned} \]

"\(k\) 阶差商" 是"\((k-1)\) 阶差商之差除以两端 \(x\) 之差"——递归定义。

定理（Newton 形式）：\(c_k = f[x_0, x_1, \ldots, x_k]\)。即

\[ \boxed{\ p(x) = \sum_{k=0}^n f[x_0, \ldots, x_k] \cdot \prod_{j=0}^{k-1}(x - x_j).\ } \]

差商有几个好性质：

  • 对称性：\(f[x_0, x_1, \ldots, x_k]\) 在重新排列节点下不变。
  • 导数关系：\(f[x_0, x_0, \ldots, x_0] = f^{(k)}(x_0) / k!\)（用 Hermite 插值或极限论证）。
  • 误差公式：\(f(x) - p(x) = f[x_0, \ldots, x_n, x] \cdot \prod_{j=0}^n (x - x_j)\)（精确，用差商 + 节点 \(x\) 的"延伸"差商）。

对角线读出系数：\(c_0 = 1, c_1 = 2, c_2 = -1.5\)。

所以 \(p(x) = 1 + 2(x - 0) - 1.5(x - 0)(x - 1) = 1 + 2x - 1.5x^2 + 1.5x = -1.5x^2 + 3.5x + 1\)。和 Lagrange 形式给的相同——多项式唯一，两条路殊途同归。

Newton 形式的大优势：

  • 加一个新点 \((x_{n+1}, y_{n+1})\)，差商表只多一行（\(O(n)\) 操作），插值多项式只多一项。增量友好。
  • 求值用嵌套形式（类似 Horner），\(O(n)\) per 求值。
  • 系数有清晰的"导数" 几何含义。

实战：自适应算法（边采样边插值，根据精度决定要不要再加点）必须用 Newton 形式。第 15 章的自适应积分会基于这个。

9.4 求值：Horner 嵌套

无论 Lagrange 还是 Newton 形式，最后你要在某个 \(x\) 上求 \(p(x)\) 的值。Newton 形式可以用嵌套计算：

\[ p(x) = c_0 + (x - x_0)\big[c_1 + (x - x_1)\big[c_2 + (x - x_2)\big[\cdots + (x - x_{n-1}) c_n\big]\cdots\big]\big]. \]

从最里面开始算：

def newton_eval(c, x_nodes, x):
    n = len(c) - 1
    result = c[n]
    for k in range(n - 1, -1, -1):
        result = c[k] + (x - x_nodes[k]) * result
    return result

每次 \(x\) 求值只 \(O(n)\) 操作。这是Horner 算法在差商形式上的对应。

Horner 不只快，还数值稳定——每步只做一次乘法和一次加法，没有幂运算的累积误差。所有多项式求值都该用 Horner，不该用直接 \(\sum a_k x^k\)（后者每个 \(x^k\) 都积累误差）。

9.5 \(n\) 大时小心：Runge 现象的预告

到这里你可能心想"插值多项式给我了一切——存在、唯一、可求、增量友好"。如果 \(n \le 10\) 左右，对的。

但 \(n\) 大时事情会出乱子。具体来说：在等距节点上对一些"看起来很温和" 的函数做高次多项式插值，插值多项式在端点附近剧烈震荡，误差完全不收敛甚至发散。这叫 Runge 现象，下一章正面讲。

预告：教科书例子是 \(f(x) = 1/(1 + 25 x^2)\) 在 \([-1, 1]\) 上。看起来是个温和的"钟形" 函数，可你用 \(n = 10, 20, 30\) 个等距节点插值，插值多项式在端点附近的震荡振幅指数级增长。\(n = 30\) 时插值多项式在端点附近最大值超过 \(10^4\)，而 \(f\) 本身从来不大于 1。

这件事的根源不是算法的错——多项式真的"通过"那 31 个节点。是多项式插值这个数学过程本身在等距节点上行为糟糕。修法在下一章：不等距取节点，把节点往端点附近聚（Chebyshev 节点）。

更通用的修法是分段低次多项式（第 11 章样条）——在每个小段里用低次（一般是 3 次），段与段在节点处光滑接合。这种"分段低次" 在工程实战里几乎是唯一可用的高维插值法。Runge 现象告诉你：用一个高次多项式覆盖整个区间是个坏主意。

下一章我们正面看 Runge 现象的原因（误差公式 + 节点几何），然后给修法（Chebyshev 节点）。这一章给的工具（Lagrange、Newton 差商）会在 Chebyshev 节点上给出非常好的结果——并不是工具本身坏，是节点选错了。
