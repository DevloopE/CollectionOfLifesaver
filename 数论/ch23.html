第 23 章 应用：现代密码学

你每次访问 https 网站，浏览器和服务器之间会做一件事：先用椭圆曲线 Diffie-Hellman（ECDH）交换出一个临时密钥，然后用这个密钥加密后续通信（用 AES-GCM）。从你按下回车到看到网页这百毫秒里，浏览器在 \(E(\mathbb F_p)\) 上做了几次群运算，服务器同样做了几次——双方算出了同一个秘密。没有一个旁观者在不解决离散对数的前提下能拿到这个秘密。

这一章把 RSA、Diffie-Hellman、ECC 这三个现代密码学的基础协议 摊开。它们每一个都基于一道数论难题——大数分解、离散对数、椭圆曲线离散对数。这一章不讲工程实现的所有细节（如那是数值方法第 23 章会更细——那本是别的角度）。这一章讲背后的数学。

23.1 公钥密码学引论：陷门函数

古典密码学（对称密码学）：通信双方共享一个密钥 \(k\)，加密 \(c = E_k(m)\)、解密 \(m = D_k(c)\)。问题——双方必须先以某种安全方式共享 \(k\)。在互联网时代，"先安全共享密钥" 这件事本身就是核心难题。

1976 年 Diffie 与 Hellman 提出公钥密码学（public-key cryptography）：每人有两个 密钥——公钥 \(\mathrm{pk}\)（公开）和私钥 \(\mathrm{sk}\)（保密）。任何人能用 Alice 的公钥加密 给 Alice 的消息；只有 Alice 用她的私钥能解密。

这种"用一把钥匙锁、另一把钥匙开" 的不对称需要一种数学结构：

\[ \boxed{\ \textbf{陷门函数}：\ f \text{ 容易算}, \ f^{-1} \text{ 极难算}, \ \textbf{除非}\text{ 你知道一个秘密 \(t\) 使 \(f^{-1}\) 容易}.\ } \]

三个具体的陷门函数（每个对应一个公钥密码体制）：

  • RSA（Rivest-Shamir-Adleman 1977）：\(f(m) = m^e \bmod n\)，陷门 = "知道 \(n\) 的素因子分解"。
  • Diffie-Hellman（1976，最古老）：实际是密钥协商 而非加密；基于"离散对数" 难。
  • 椭圆曲线密码（ECC，Koblitz / Miller 1985）：基于椭圆曲线上的离散对数难。

下面三节具体看每一个。

23.2 RSA：基于"大数分解难"

RSA 是公钥密码学最经典的协议。设置如下：

密钥生成（Alice 一次性做）。

  1. 选两个大素数 \(p, q\)（实践中各 1024 位以上，所以 \(n = pq\) 是 2048 位）。
  2. 算 \(n = pq\) 与 \(\varphi(n) = (p-1)(q-1)\)（链回ch6 Euler 函数）。
  3. 选 \(e\) 满足 \(\gcd(e, \varphi(n)) = 1\)（实践中常用 \(e = 65537 = 2^{16} + 1\)，因 \(e\) 是素数且二进制只有两个 1，模幂运算快）。
  4. 算 \(d = e^{-1} \bmod \varphi(n)\)（用 Euclid 扩展算法，链回ch2）。
  5. 公钥：\(\mathrm{pk} = (n, e)\)。私钥：\(\mathrm{sk} = (n, d)\)。抛弃 \(p, q, \varphi(n)\)（它们的存在会泄露私钥）。

加密（Bob 给 Alice 发送 \(m\)，需 \(0 \le m < n\)）。

\[ c = m^e \bmod n. \]

解密（Alice 拿到 \(c\)）。

\[ m = c^d \bmod n. \]

正确性证明。需要证 \((m^e)^d \equiv m \pmod n\)。

\(ed \equiv 1 \pmod{\varphi(n)}\)，所以 \(ed = 1 + k \varphi(n)\) 某 \(k \ge 1\)。所以

\[ m^{ed} = m^{1 + k \varphi(n)} = m \cdot (m^{\varphi(n)})^k. \]

由 Euler 定理（链回ch6）：当 \(\gcd(m, n) = 1\)，\(m^{\varphi(n)} \equiv 1 \pmod n\)。所以 \(m^{ed} \equiv m \pmod n\)。\(\checkmark\)

边界情况 \(\gcd(m, n) > 1\)：此时 \(p \mid m\) 或 \(q \mid m\)。但 \(m < n = pq\)，所以不会同时 \(p \mid m\) 和 \(q \mid m\)。设 \(p \mid m\)，则 \(m \equiv 0 \pmod p\)，所以 \(m^{ed} \equiv 0 \equiv m \pmod p\)。另一方面 \(\gcd(m, q) = 1\)，所以 Euler 定理给 \(m^{ed} \equiv m \pmod q\)。CRT（链回ch7）合并：\(m^{ed} \equiv m \pmod n\)。完整正确性 \(\checkmark\)

安全性。RSA 的安全建立在分解 \(n\) 难。若你能分解 \(n = pq\)，立即算出 \(\varphi(n) = (p-1)(q-1)\)，然后算 \(d = e^{-1} \bmod \varphi(n)\)——破解。反过来也大致正确：若你能解密任何 RSA 密文（即破解 RSA），则你能分解 \(n\)（这是条件 等价，1985 年由 Rabin 与 Coppersmith 证明的更精确版本）。

分解 \(n\) 的最快经典算法（数域筛 GNFS，1990 年代）复杂度 \(\exp(O((\log n)^{1/3} (\log\log n)^{2/3}))\)——次指数。它比"试除每个 \(p \le \sqrt n\)" 的 \(O(\sqrt n)\) 快得多，但仍然亚指数——\(n\) 加倍位数，运行时间不是翻倍 而是大约 \(2^{1/3}\) 倍。所以 2048 位 RSA 在 2025 年仍安全（最佳已知分解记录是 829 位 \(n\)，2020 年由学术合作完成、用了 2700 CPU-年）。

历史小注。1973 年英国 GCHQ 的 Cocks 内部已经提出过本质相同的算法，但因为情报机密直到 1997 年才解密。RSA 三人独立发明并 1977 年公开发表——所以学术界称 "RSA"，密码学历史承认 Cocks 的优先权。

23.3 Diffie-Hellman 密钥交换

RSA 解决"怎么从公钥推私钥难"。Diffie-Hellman 解决一个不同的问题：双方在公开信道上协商出一个共享密钥，使旁观者算不出。

协议（Alice 与 Bob 协商）。

  1. 双方公开同意一个大素数 \(p\) 与一个原根 \(g \in (\mathbb Z/p)^\times\)（链回ch8原根）。
  2. Alice 选私钥 \(a \in \{1, \ldots, p - 2\}\)，发送 \(A = g^a \bmod p\) 给 Bob。
  3. Bob 选私钥 \(b\)，发送 \(B = g^b \bmod p\) 给 Alice。
  4. 双方计算共享密钥： \[ K = A^b = B^a = g^{ab} \bmod p. \] Alice 用她的 \(a\) + Bob 发的 \(B\)；Bob 对称做。

旁观者看到 \(p, g, A, B\) 但不知道 \(a, b\)。他要算 \(K\) 必须从 \(A = g^a\) 推出 \(a\)（或从 \(B\) 推 \(b\)）——这是离散对数问题（DLP）：

\[ \boxed{\ \text{DLP}：\ \text{给 } g, A = g^a \bmod p, \text{ 求 } a.\ } \]

这是ch8提到的"原根难题"——给一个原根 \(g\)，计算 \(g^x \mod p\) 是 \(O(\log p)\) 步快速幂；反过来 \(g^x = A\) 求 \(x\) 没有已知多项式时间算法。最好的经典算法（数域筛、index calculus）复杂度亚指数——和分解类似但有所不同。

例（玩具规模）。取 \(p = 23\)、\(g = 5\)（5 是 mod 23 的原根，因 \(\mathrm{ord}_{23}(5) = 22 = \varphi(23)\)）。Alice 选 \(a = 6\)，发 \(A = 5^6 \bmod 23 = 15625 \bmod 23 = 8\)。Bob 选 \(b = 15\)，发 \(B = 5^{15} \bmod 23\)；算 \(5^4 = 625 \equiv 4\)，\(5^8 \equiv 16\)，\(5^{15} = 5^8 \cdot 5^4 \cdot 5^2 \cdot 5 \equiv 16 \cdot 4 \cdot 2 \cdot 5 = 640 \equiv 640 - 27 \cdot 23 = 640 - 621 = 19\)。所以 \(B = 19\)。共享密钥 \(K = B^a = 19^6 \bmod 23\)（Alice 算）= \(A^b = 8^{15} \bmod 23\)（Bob 算）。两个应该相等。具体算 \(19^2 = 361 \equiv 361 - 15 \cdot 23 = 361 - 345 = 16\)；\(19^4 \equiv 16^2 = 256 \equiv 256 - 11 \cdot 23 = 256 - 253 = 3\)；\(19^6 \equiv 19^4 \cdot 19^2 \equiv 3 \cdot 16 = 48 \equiv 48 - 23 = 2\)。所以 \(K = 2\)。Bob 应该算出同样：\(8^{15} \bmod 23\)；\(8 = 5^6\)，所以 \(8^{15} = 5^{90} = 5^{90 \bmod 22} = 5^2 = 25 \equiv 2 \pmod{23}\)。\(\checkmark\)

实战使用 2048 位 \(p\)（实际产品用 RFC 3526 给定的 2048-3072 位的"safe primes" \(p = 2q + 1\)，\(q\) 也素）。

DH vs RSA。RSA 既能加密也能签名；DH 只是密钥协商，协商完密钥后续仍要用对称密码（如 AES）。但 DH 有一个优点叫前向安全性（forward secrecy）：如果 Alice 每次会话生成新的临时 \(a\)（"临时 DH" / Ephemeral DH），即使将来 Alice 的长期私钥被攻破，过去的会话密钥 仍安全（因为它们只依赖于已经被销毁的临时 \(a\)）。这是为什么 https 现在不用 RSA 做密钥交换，而用 ECDHE（椭圆曲线临时 DH）。

23.4 椭圆曲线密码 ECC

DH 在 \((\mathbb Z/p)^\times\) 这个乘法群里做。把这个群换成ch19椭圆曲线 \(E(\mathbb F_p)\) 上的群，得到椭圆曲线 Diffie-Hellman（ECDH）。

记住ch19的核心：椭圆曲线 \(E : y^2 = x^3 + ax + b\) 上的有理点构成阿贝尔群，群运算是几何的"三点共线和为零" 法则——但这个法则在 \(\mathbb F_p\) 上同样起作用。所以 \(E(\mathbb F_p)\) 是有限阿贝尔群。Hasse 界（链回ch19）给 \(\#E(\mathbb F_p) \approx p\)（精确：\(|\#E(\mathbb F_p) - (p + 1)| \le 2\sqrt p\)）。

ECDH 协议。

  1. 双方公开同意椭圆曲线 \(E\)、有限域 \(\mathbb F_p\)、与一个"基点" \(G \in E(\mathbb F_p)\)（高阶点，常 \(\mathrm{ord}(G) = n\) 是大素数）。
  2. Alice 选私钥 \(a \in \{1, \ldots, n-1\}\)，发送公钥 \(A = a G \in E(\mathbb F_p)\)（"\(a\) 倍点"）。
  3. Bob 选 \(b\)，发 \(B = b G\)。
  4. 共享密钥 \(K = a B = b A = (ab) G\)。

旁观者面对的难题：从 \(G\)、\(A = aG\) 算 \(a\)。这是椭圆曲线离散对数问题（ECDLP）：

\[ \boxed{\ \text{ECDLP}：\ \text{给 } G, P = aG \in E(\mathbb F_p), \text{ 求 } a.\ } \]

关键事实：ECDLP 没有亚指数算法（不像 DLP 与分解）。最佳已知算法是Pollard rho，复杂度 \(O(\sqrt n)\)（\(n\) 是基点的阶）。这是真正的指数——256 位的 \(n\)（即 \(\sqrt n \approx 2^{128}\) 步）足以抵抗量子前的所有攻击。

对比：

  • RSA 2048 位 \(\approx\) ECDLP 224 位 \(\approx\) 112 位对称安全。
  • RSA 3072 位 \(\approx\) ECDLP 256 位 \(\approx\) 128 位对称安全。

ECC 的密钥短得多——这是为什么移动设备、物联网、嵌入式系统几乎全用 ECC 而不是 RSA。具体的，TLS 1.3（2018+）默认用 X25519（一种特殊曲线 \(y^2 = x^3 + 486662 x^2 + x\) 在 \(\mathbb F_{2^{255} - 19}\) 上的形式，由 Bernstein 2005 设计）做密钥交换，签名用 Ed25519。

具体曲线选择。NIST 1999 推荐曲线 P-256（也叫 secp256r1）：

\[ y^2 = x^3 - 3x + b, \quad p = 2^{256} - 2^{224} + 2^{192} + 2^{96} - 1. \]

这条曲线在 https / SSH / Bitcoin 之外的几乎所有商业 系统里使用。Bitcoin 用 secp256k1（\(b\) 不同）。Curve25519 / Ed25519 因 Bernstein 设计的"常数时间安全实现" 在学术界与新一代协议（Signal、WireGuard、SSH 默认）里更受推崇。

所有这些曲线都满足"阶 \(\#E(\mathbb F_p) = h \cdot n\)" 的形式，\(n\) 是大素数（256 位以上），\(h \in \{1, 2, 4, 8\}\) 是cofactor。基点 \(G\) 选阶 \(n\)。这样 ECDLP 有 \(\sqrt n \approx 2^{128}\) 的安全度。

23.5 量子计算威胁与 Shor 算法

RSA、DH、ECC 三个体系的共同基础：分解或离散对数对经典计算机难。但它们对量子计算机 怎样？

1994 年 Peter Shor 给出震动密码学界的算法：量子计算机能在多项式时间内分解整数与算离散对数。具体的：

\[ \boxed{\ \text{Shor 算法分解 \(n\)：复杂度 } O((\log n)^3) \text{（量子）}.\ } \]

核心思想：分解 \(n\) 等价于"找一个数 \(a\) 满足 \(\mathrm{ord}_n(a)\) 偶" 然后从这个阶推出非平凡因子。Shor 用量子 Fourier 变换（QFT，链回复分析中 Fourier）在 \(\mathbb Z/N\) 上找周期——这是 QFT 在量子机上多项式时间能做的事，但经典机上没人知道怎么做到多项式时间。

具体步骤：

  1. 选 \(a < n\) 与 \(n\) 互素，量子算 \(\mathrm{ord}_n(a)\)（QFT 找 \(f(x) = a^x \bmod n\) 的周期）。
  2. 若阶 \(r\) 偶且 \(a^{r/2} \not\equiv -1 \pmod n\)，则 \(\gcd(a^{r/2} - 1, n)\) 与 \(\gcd(a^{r/2} + 1, n)\) 给 \(n\) 的非平凡因子。
  3. 否则换 \(a\) 重试。

所有公钥密码学都倒。Shor 算法的简单变体也破 DLP（链回ch8）和 ECDLP——同样是"QFT 找周期" 套路在合适的群上。

2025 年的现状：能做 Shor 的量子计算机原型 已存在（IBM 2023 演示 433-qubit 处理器，Google 2024 演示 70-qubit Willow），但分解有意义大小（≥1024 位）的 \(n\) 需要数百万 qubit + 强容错纠错。密码相关量子机（CRQC）的预期到来时间是 2030-2040 年，但精确时间不可预知。

这条威胁是真的——所有今天用 RSA / DH / ECC 加密的历史数据，被存档下来后将在 CRQC 来到时被解密。这条逻辑叫""采集"现在 / 解密未来"（harvest now, decrypt later）——所以即使 CRQC 在 10-15 年才来，今天的 长期敏感数据（医疗、外交、商业秘密）已经处于威胁中。这就是后量子密码学 的紧迫性。

23.6 后量子密码学

"后量子密码学"（Post-Quantum Cryptography, PQC）= 设计对量子也安全 的密码协议。它的数学基础不能 是分解或离散对数（因为 Shor 破它们）。已知对量子也难 的问题：

  • 格上短向量问题（SVP）+ 带误差学习（LWE）。这是 PQC 主流。
  • 编码理论问题（McEliece 1978，最古老的 PQC 候选，没人破过但密钥很大）。
  • 多变量多项式系统（基于解 \(n\) 元二次方程组难）。
  • 哈希签名（基于哈希函数的"找碰撞" 难，量子也只能 Grover 加速到 \(\sqrt n\)，不致命）。
  • 同源椭圆曲线（SIDH/SIKE，\(\to\) 2022 年被经典攻破，已废弃）。

NIST 从 2016 起做"PQC 标准化竞赛"。2022 年 7 月公布第一轮选定的算法：

  • Kyber（密钥协商 / KEM）—— 基于模 LWE。
  • Dilithium（数字签名）—— 基于模 LWE。
  • SPHINCS+（哈希签名）—— 基于哈希。
  • Falcon（签名）—— 基于NTRU 格。

2024 年 8 月 NIST 发布正式标准 FIPS 203（Kyber）、FIPS 204（Dilithium）、FIPS 205（SPHINCS+）。

带误差学习问题（LWE，Regev 2005）的简化陈述：在 \(\mathbb Z/q\) 上选 secret \(s \in (\mathbb Z/q)^n\)。给攻击者一系列样本 \((a_i, b_i)\)，其中 \(a_i \in (\mathbb Z/q)^n\) 随机、\(b_i = \langle a_i, s \rangle + e_i \bmod q\)，\(e_i\) 是小的随机误差。问题：恢复 \(s\)。

这等价于解噪音线性方程组。Regev 证明这条问题"对量子也难"——更精确：把 LWE 解到任意精度归约到 解格上的最坏情形 短向量问题，而后者没有已知量子算法（包括 Shor）能解。

Kyber 与 Dilithium 都是 LWE 的结构化 版本（Module-LWE，把 \((\mathbb Z/q)^n\) 换成多项式环 \((\mathbb Z/q)[x]/(x^d + 1)\) 上的模），这样密钥短得多（千字节级而不是兆字节）。

2025 年现状：主要浏览器（Chrome / Firefox / Safari）已经支持混合 TLS——同时用 X25519（经典 ECC）+ Kyber（PQC），双重保护。Cloudflare、Google 服务器端已 100% 部署。整个互联网的"大迁移到 PQC" 预计 2025-2035 年完成。

23.7 实战：libsodium / OpenSSL 的具体使用

本节超快过一遍写代码时怎么调这些算法。这本书是数学书不是工程书——细节请查官方文档。但你应该知道不要自己实现：

  • 密码学的工程实现极易写错（侧信道、随机数生成器、constant-time 操作）。从不自己写 RSA / ECC——总用经过审计的库。
  • "roll your own crypto" 是密码学最有名的反模式。学术界已经看过太多次"新发明的密码协议" 第二天被攻破的案例。

libsodium（基于 NaCl，Bernstein 设计）。最容易用的现代库：

// Alice: 生成密钥对
unsigned char pk[crypto_box_PUBLICKEYBYTES];
unsigned char sk[crypto_box_SECRETKEYBYTES];
crypto_box_keypair(pk, sk);   // Curve25519 + Salsa20 + Poly1305

// Bob 用 Alice 的 pk 加密
unsigned char ciphertext[CIPHERTEXT_LEN];
crypto_box(ciphertext, message, message_len, nonce, alice_pk, bob_sk);

// Alice 用她的 sk 解密
crypto_box_open(plaintext, ciphertext, ciphertext_len, nonce, bob_pk, alice_sk);

这一段代码下面发生的是：Curve25519 ECDH + 派生对称密钥 + XSalsa20 加密 + Poly1305 认证。你看不见 这些数学细节——库都帮你做了，并且做得"常数时间" 不泄露侧信道。

OpenSSL。更老但用得最广。它的 API 复杂——上面 libsodium 的 4 行变成 OpenSSL 的 50 行。除非你必须兼容现有 PKI 否则建议用 libsodium 或 Go 的 crypto/... 系列。

其它选择：Rust ring、RustCrypto 系列；Go crypto/elliptic、crypto/ed25519；Python cryptography 库。不要用 Python 标准库的 secrets 之外的密码学（标准库只有 secrets 是设计来给生产用的；random、\(\mathrm{md5}\) 等都是不安全的）。

23.8 数论 + 密码学的未来

密码学是数论应用最直接的领域。每个新的数论结构都立刻 引出新的密码协议尝试：

  • 同源密码（isogeny-based crypto）：基于"椭圆曲线之间的同源映射"。SIDH 2011 提出，2022 年由 Castryck-Decru 经典攻破——这是个大教训：新的数论结构需要至少 10 年的同行审计 才能被信任。
  • 双线性配对（bilinear pairings）：椭圆曲线上的 Weil / Tate 配对（链回ch19）让"身份基加密"（identity-based encryption）成为可能——"用 Alice 的电子邮件地址作为公钥"。这是 Boneh-Franklin 2001 的发明。
  • 零知识证明（ZKP）+ SNARKs / STARKs：基于多项式承诺、椭圆曲线配对、哈希函数的复合结构。是 Web3 / 区块链可扩展性的核心工具（Ethereum 二层用大量 ZK Rollups）。
  • 全同态加密（FHE）：基于 LWE 的格密码。允许对加密数据直接做计算——"云服务商在不解密的前提下处理你的数据"。Gentry 2009 的发明，2025 年已经有了 IBM HElib、Microsoft SEAL 等可用库。

所有这些方向都 是数论 + 代数几何 + 表示论的具体应用。你这本书的每一章都在某种程度上对应一种现代密码工具的数学基础——从最基础的 \(\mathbb Z/n\)（RSA、DH）到最尖端的椭圆曲线配对（IBE、ZKP）和格（FHE、PQC）。

请你停下来感受这件事：

  • 1801 年 Gauss 的《算术研究》 是一本"纯数学" 著作，没有任何应用动机。
  • 1976 年 Diffie-Hellman 把 Gauss 的"原根" 变成密钥协商。
  • 1985 年 Koblitz / Miller 把椭圆曲线（200 年前 Abel、Jacobi 的纯研究）变成短密钥密码。
  • 2024 年 NIST FIPS 203 把 Regev 2005 的纯数学 LWE 问题变成互联网标准。

"纯数学" 与"实用" 之间的距离在数论里通常是 30-200 年——但每隔几十年总有一次令人惊讶的转化。这本书前 22 章那些看似"没有应用" 的结构（类群、模形式、Galois 表示、椭圆曲线 \(L\)-函数），它们的未来应用可能是下一个 30 年的密码学。Hardy 1940 写"数论的纯粹是它的优点之一—— 它将永远没用"——他错得彻底。

下一章是这本书的最后一章。我们用整本书前 23 章的工具 拼出 Wiles 1995 证 Fermat 大定理的全图景——把"整数 \(\mathbb Z\)" 这个最朴素的对象，看到它复杂得需要 358 年人类最聪明的头脑 + 几乎所有 20 世纪现代数学的故事的尾声。
