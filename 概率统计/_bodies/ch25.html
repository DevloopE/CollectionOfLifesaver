第 25 章 信息论入门

"信息"是个我们日常天天用却很少认真定义的词。手机通了一条短信"今天下雨"，信息量大不大？取决于：在沙漠里，这是天大的新闻；在天天下雨的城市，这几乎啥也没说。信息的量似乎跟"意外程度"有关——越是不那么意料之中的消息，"信息"就越多。这条朴素直觉，由 1948 年 Claude Shannon 在那篇划时代的论文《A Mathematical Theory of Communication》中正式数学化，开辟了一门叫做信息论（information theory）的学科。

本章不是 Shannon 著作的精确再现（那需要一整本书），但本章会让你认识信息论的几位核心明星——自信息、熵、条件熵、互信息、KL 散度——并初步了解它们如何决定通信、压缩、机器学习的极限。最重要的：你将看到现代深度学习里到处出现的"交叉熵损失"，本质上就是 KL 散度的化身——分类器的训练，从信息论看，就是把模型分布拉得最贴近真实分布。

25.1 信息的量化需求：通信、压缩、ML

为什么需要把"信息"量化？三个动机。

通信：你要把一段消息从甲传到乙——信道的带宽有限、有噪声。一秒钟能稳定传多少 bit？这个上限叫信道容量，由互信息决定（25.9 节）。Shannon 的天才在于他先把"信息量"用一个数学对象（熵）量化了，然后才能说"通信速率不能超过容量"——这是信息论第一定理的内容。

压缩：一段文本可以压缩到多小？英文有大量冗余（比如 "q" 后面几乎一定是 "u"），所以可压缩。最优无损压缩的极限是多少？答案：等于信源的熵。这就是 Shannon 源编码定理（25.8 节）。

机器学习：训练一个分类器去给图片打标签——本质上是在学一个分布 \(q(y \mid x)\) 去逼近真实分布 \(p(y \mid x)\)。"逼近得好不好"用什么度量？最常用的就是交叉熵（cross-entropy）——它是 KL 散度加上一个常数，跟 KL 散度等价（25.10 节）。所以现代深度学习中训练模型，从信息论看，就是最小化模型分布与真实分布的 KL 散度。

这三件事看起来毫不相关——通信工程师、压缩算法设计师、机器学习研究员各自有各自的目标——但它们的数学根都是同一个：用一组明确定义的量化指标处理"信息"。这就是信息论的统一力量。

25.2 自信息 \(I(x) = -\log p(x)\)

从最朴素的问题出发：单个事件"\(X = x\)"发生，承载多少信息？

从直觉上要满足两条：

  • (1) 事件越罕见——\(p(x)\) 越小——它"发生"时给我们带来的信息越多。"今天太阳从东方升起" 概率几乎为 1，信息量为 0；"小行星撞月球" 概率极小，信息量极大。所以信息量应该是 \(p\) 的递减函数。
  • (2) 两个独立事件"都发生"的总信息量，应该等于"各自发生"的信息量之和——加和性。这是个极其自然的要求：你听到"我中彩票了"和"我升职了"两条独立好消息，总惊喜是两份惊喜的和。

条件 (2) 用数学说："如果 \(X\) 与 \(Y\) 独立——即 \(p(x, y) = p(x) p(y)\)——则 \(I(x, y) = I(x) + I(y)\)"。要把 \(p(x) p(y)\) 这种乘积变成加法，唯一自然的运算就是对数。

结合两条要求，定义事件 \(X = x\) 的自信息（self-information，又叫信息量、surprisal）为

\[ \boxed{I(x) = -\log p(x).} \]

这里的对数底数有讲究：

  • 底数为 2（最常用）：单位是比特（bit）。例如均匀硬币正面朝上的事件 \(p = 1/2\)，\(I = -\log_2(1/2) = 1\) bit。
  • 底数为 \(e\)（自然对数）：单位是奈特（nat）。在数学推导（出现 \(\ln\) 时）方便。
  • 底数为 10：单位是哈特（hartley）。已经少用。

不同底数只是改一个常数倍，本质相同。本书后面默认 \(\log = \log_2\)（除非另有说明）。

请你注意自信息是关于事件的——它依赖于具体取值 \(x\)。不同取值的事件信息量不同。下一节我们对这件事取期望，得到关于随机变量的"平均信息量"——这就是熵。

25.3 香农熵 \(H(X)\)

设 \(X\) 是个离散随机变量，取值在某有限集 \(\mathcal{X}\) 中，分布 \(p(x) = P(X = x)\)。把每个取值的自信息按其概率加权平均，就得到 \(X\) 的香农熵（Shannon entropy）：

\[ \boxed{H(X) = \mathbb{E}[I(X)] = -\sum_{x \in \mathcal{X}} p(x) \log p(x).} \]

约定：\(0 \log 0 = 0\)（极限角度看 \(p \log p \to 0\) 当 \(p \to 0\)）。

熵衡量的是"\(X\) 的不确定性"——你在抽 \(X\) 之前对结果的"无知程度"。极端例子：

  • \(X\) 是个常数（取某值的概率为 1，其余为 0）：完全没有不确定性，\(H(X) = 0\)。
  • \(X\) 在 \(n\) 个取值上均匀分布（每个 \(1/n\)）：不确定性最大，\(H(X) = \log n\)。这就是为什么熵的单位是 bit——一个均匀的二态变量（公正硬币）熵为 1 bit。

事实上对所有离散分布 \(p\) 在 \(n\) 个值上，

\[ 0 \leq H(X) \leq \log n, \]

左边等号当 \(p\) 集中于一个值时取到，右边等号当 \(p\) 均匀时取到。"均匀分布最不确定"——这是人人都有的直觉，香农熵把这条直觉数学化了。

例 25.1（计算几个分布的熵）　设 \(X \sim \mathrm{Bernoulli}(p)\)：取 1 的概率 \(p\)，取 0 的概率 \(1 - p\)。熵

\[ H(X) = -p \log p - (1 - p) \log(1 - p). \]

这条函数叫二元熵函数，记作 \(H(p)\)。\(H(0) = H(1) = 0\)（确定性），\(H(1/2) = 1\) bit（最大不确定性）。它在 \([0, 1]\) 上是凹的、对称的钟形。

例 25.2（公正与不公正硬币）　公正硬币 \(p = 1/2\)：\(H = 1\) bit。不公正硬币 \(p = 0.9\)：\(H = -0.9 \log_2 0.9 - 0.1 \log_2 0.1 \approx 0.469\) bit。"不那么意料之外"的硬币熵小——这与直觉吻合：你"预测下一次结果"时，若硬币偏，几乎肯定可以准；若公正，蒙的概率只有一半。

25.3.1 熵的连续推广：微分熵

对连续随机变量 \(X\) 密度 \(f(x)\)，定义类似

\[ h(X) = -\int f(x) \log f(x) \, dx, \]

叫做微分熵（differential entropy）。请注意它跟离散熵有重要差别：微分熵可以取负值（因为 \(f(x)\) 可大于 1），它在坐标变换下不像离散熵那样"绝对"。微分熵在通信信道分析中仍然很重要（特别在 Gaussian 信道），但用它时要小心解读。

25.4 联合熵与条件熵

对两个随机变量 \(X, Y\) 联合分布 \(p(x, y)\)，联合熵定义为

\[ H(X, Y) = -\sum_{x, y} p(x, y) \log p(x, y). \]

它衡量"\(X\) 和 \(Y\) 一起的不确定性"。

给定 \(X = x\) 后 \(Y\) 的条件分布是 \(p(y \mid x)\)，那一条件下 \(Y\) 的熵是 \(H(Y \mid X = x) = -\sum_y p(y \mid x) \log p(y \mid x)\)。把这个量按 \(X\) 的分布取期望，得到条件熵：

\[ \boxed{H(Y \mid X) = \sum_x p(x) H(Y \mid X = x) = -\sum_{x, y} p(x, y) \log p(y \mid x).} \]

它衡量"已知 \(X\) 后 \(Y\) 还残余的不确定性"。

关键关系——熵的链式法则：

\[ \boxed{H(X, Y) = H(X) + H(Y \mid X) = H(Y) + H(X \mid Y).} \]

用人话解释：联合不确定性 = "先知道 \(X\) 的不确定性"+"知道 \(X\) 后 \(Y\) 还有的不确定性"。这条法则形式上跟条件概率的链式法则 \(p(x, y) = p(x) p(y \mid x)\) 完全平行——取对数即可看出来。

另一关键关系：条件减少不确定性——

\[ H(Y \mid X) \leq H(Y), \]

等号当且仅当 \(X\) 与 \(Y\) 独立。"知道一些总比不知道多" 的数学化。

25.5 互信息 \(I(X; Y)\)

"\(X\) 提供的关于 \(Y\) 的信息量"是多少？答：\(Y\) 的不确定性减去给 \(X\) 后 \(Y\) 的不确定性——

\[ \boxed{I(X; Y) = H(Y) - H(Y \mid X).} \]

这就是互信息（mutual information）。它衡量两个变量"共享"多少信息。几个等价表达式（都很有用）：

\[ I(X; Y) = H(X) + H(Y) - H(X, Y), \] \[ I(X; Y) = \sum_{x, y} p(x, y) \log \frac{p(x, y)}{p(x) p(y)}. \]

第一种说"互信息 = 各自不确定性之和 - 联合不确定性"——重叠部分。第二种把它写成 KL 散度的形式（下一节我们就引入 KL 散度）：互信息正是"联合分布"和"两个边际分布的乘积"之间的 KL 距离。

关键性质：

  • 对称：\(I(X; Y) = I(Y; X)\)。
  • 非负：\(I(X; Y) \geq 0\)，等号当且仅当 \(X\) 与 \(Y\) 独立。
  • "\(X\) 知道一切关于 \(Y\) 的信息"时（比如 \(Y = f(X)\)），\(I(X; Y) = H(Y)\)。

互信息是机器学习里许多目标函数的真正本质——比如"特征选择"问题"找跟标签 \(Y\) 互信息最大的特征 \(X\)"，"信息瓶颈"原则"找一个隐表示 \(Z\) 使 \(I(X; Z)\) 小但 \(I(Z; Y)\) 大"。它在通信里就是信道容量的核心（25.9 节）。

例 25.3（两枚硬币的互信息）　两枚公正硬币 \(X, Y\) 独立。\(H(X) = H(Y) = 1\) bit，\(H(X, Y) = 2\) bit，\(I(X; Y) = 1 + 1 - 2 = 0\) ✓（独立时互信息为 0）。

现在改为 \(Y = X\)（两枚硬币同步）。\(H(X, Y) = H(X) = 1\)，\(I(X; Y) = 1 + 1 - 1 = 1\) bit——\(X\) 完全决定 \(Y\)，所以互信息等于 \(H(Y) = 1\) bit。

25.6 KL 散度 \(D_{KL}(P \| Q)\)

给定同一样本空间上的两个分布 \(P\) 与 \(Q\)，KL 散度（Kullback-Leibler divergence，又叫"相对熵"）定义为

\[ \boxed{D_{KL}(P \| Q) = \sum_x p(x) \log \frac{p(x)}{q(x)}.} \]

（连续情形把求和换成积分。约定 \(0 \log 0 = 0\)；如果某个 \(x\) 处 \(p(x) > 0\) 但 \(q(x) = 0\)，则 \(D_{KL} = +\infty\)。）

它度量"用分布 \(Q\) 来编码服从 \(P\) 的数据时，比用 \(P\) 自己编码多花的平均比特数"。下一节会从源编码角度解释这一直观。

关键性质：

  • 非负：\(D_{KL}(P \| Q) \geq 0\)，等号当且仅当 \(P = Q\)（25.7 节用 Jensen 不等式证）。这就是为什么它常被称为"分布之间的距离"——尽管它不是真正的距离（不对称、不满足三角不等式）。
  • 不对称：\(D_{KL}(P \| Q) \neq D_{KL}(Q \| P)\) 一般情形。这反映了"用 \(Q\) 近似 \(P\)" 和"用 \(P\) 近似 \(Q\)" 是不同的代价。

请你注意这一不对称性——它是 KL 散度最容易被新手误解的地方。比如 \(P\) 在某处很大、\(Q\) 在那里几乎为 0，则 \(D_{KL}(P \| Q)\) 极大（\(P\) 的支撑必须被 \(Q\) "覆盖"住）；反过来 \(D_{KL}(Q \| P)\) 可能不大。这不对称在变分推断里造成"模式搜寻"vs"模式覆盖"的不同行为，是个有趣的应用细节。

互信息和 KL 散度的关系前面已经提到：

\[ I(X; Y) = D_{KL}(p(x, y) \| p(x) p(y)). \]

所以互信息度量"联合分布偏离独立分布"的程度。

25.7 KL 散度非负 — 用 Jensen 不等式证

这是信息论里最基本、最被反复使用的不等式。证明非常优雅，请你跟着走一遍。

第 11 章我们见过 Jensen 不等式：对凸函数 \(\varphi\) 和随机变量 \(X\)，\(\mathbb{E}[\varphi(X)] \geq \varphi(\mathbb{E}[X])\)；对凹函数则反向。\(\log\) 是凹函数，所以 \(\mathbb{E}[\log Y] \leq \log \mathbb{E}[Y]\)。

现在

\[ - D_{KL}(P \| Q) = \sum_x p(x) \log \frac{q(x)}{p(x)} = \mathbb{E}_{X \sim P}\left[\log \frac{q(X)}{p(X)}\right]. \]

对凹函数 \(\log\) 用 Jensen：

\[ \mathbb{E}_{X \sim P}\left[\log \frac{q(X)}{p(X)}\right] \leq \log \mathbb{E}_{X \sim P}\left[\frac{q(X)}{p(X)}\right] = \log \sum_x p(x) \cdot \frac{q(x)}{p(x)} = \log \sum_x q(x) = \log 1 = 0. \]

所以 \(-D_{KL} \leq 0\)，即 \(D_{KL} \geq 0\)。等号当且仅当 \(q(x)/p(x)\) 在 \(P\) 几乎处处取常数——结合两者都积分为 1，这只能是 \(p \equiv q\)。证毕。

这条证明的"力气"全在 Jensen 不等式上——\(\log\) 的凹性是核心。从这条证明你能看出，"KL 非负"本质上是"算术-几何均值不等式"在分布上的化身。

25.8 香农源编码定理：熵是无损压缩的极限

有了熵的定义，我们可以问：一段服从分布 \(p\) 的随机消息，平均最少需要多少 bit 才能无损地编码？

Shannon 给出的答案漂亮极了：

对任何无损编码方案，平均码长 \(L \geq H(X)\)；并且存在编码方案使 \(L \to H(X)\)（任意接近）。

这就是源编码定理（source coding theorem）。它说熵就是无损压缩的下限——再聪明的算法也压不到熵以下。这是为什么"熵"被解读为"信源的真正信息量"。

例：英文 26 个字母 + 空格，按均匀分布每符号需 \(\log_2 27 \approx 4.75\) bit；但英语字母分布远不均匀（"e" 频率高、"z" 频率低），加上字母间相关性，英文文本的实际熵约为 1-1.5 bit/字符——这就是为什么 zip、gzip 能把英文文本压缩到原大小的 30% 左右。

具体的实现：霍夫曼编码（Huffman coding）给概率高的符号短码、概率低的符号长码，渐近达到熵；算术编码（arithmetic coding）能把整段消息编成一个数，比霍夫曼更接近熵；LZ 系列（zip 用的）通过查找重复模式压缩，不需要预先知道分布，对实际文件极有效。

对有损压缩（如 JPEG、MP3），有更弱的下限——叫"率失真函数"——本章不展开，但你应该知道它存在。

25.9 信道容量 \(C = \max_{p(x)} I(X; Y)\)

"通信信道"是个把输入 \(X\) 映为输出 \(Y\) 的东西，可能有噪声——比如二元对称信道（输入 0 或 1，每个 bit 以概率 \(\epsilon\) 翻转）。我们想知道：每秒能稳定传多少 bit 的信息？

Shannon 的信道编码定理给出答案：通过这条信道每次使用，能稳定传输的信息率不超过

\[ \boxed{C = \max_{p(x)} I(X; Y),} \]

叫信道容量——是"最优输入分布下，输入与输出的互信息"。并且只要传输率 \(< C\)，就存在编码方案使错误率任意小；只要 \(> C\)，错误率必下界离零。

这条定理是信息论的"第二定理"——它告诉你噪声并不阻止可靠通信，只是限制速率。这是 1940 年代极其反直觉的结论：当时通信工程师以为噪声会不可避免地引入误差，无论多慢的速率也躲不开；Shannon 证明只要速率小于容量，就有"编码魔法"让错误率趋于零。

例（二元对称信道，错率 \(\epsilon\)）：容量 \(C = 1 - H(\epsilon)\) bit/use。\(\epsilon = 0\)（无噪声）时 \(C = 1\)（无损通信）；\(\epsilon = 1/2\)（完全随机翻转）时 \(C = 0\)（信道完全没用）；\(\epsilon = 0.1\) 时 \(C = 1 - H(0.1) \approx 0.531\) bit。

实现"达到容量"的具体编码（Turbo 码、LDPC 码、Polar 码等）是工程奇迹——它们让 5G、Wi-Fi、磁盘存储以接近 Shannon 极限的效率工作。

25.10 信息论与 ML：交叉熵损失 = KL + 熵

本章最后一站：把信息论的工具放到机器学习的语境里。这是这门学科最现代的应用之一。

分类问题的设定：输入 \(x\)，真实标签 \(y\) 的分布 \(p(y \mid x)\)；模型输出预测分布 \(q_\theta(y \mid x)\)。"损失函数"度量 \(q\) 跟 \(p\) 偏差多大。最常用的就是交叉熵损失（cross-entropy loss）：

\[ H(p, q) = -\sum_y p(y) \log q(y). \]

注意它跟"熵 \(H(p)\)" 形式上很像但参数不同——熵是 \(-\sum p \log p\)，交叉熵是 \(-\sum p \log q\)。直接展开：

\[ H(p, q) = -\sum_y p(y) \log q(y) = -\sum_y p(y) \log p(y) + \sum_y p(y) \log \frac{p(y)}{q(y)} = H(p) + D_{KL}(p \| q). \]

这是个非常关键的等式：

\[ \boxed{H(p, q) = H(p) + D_{KL}(p \| q).} \]

"交叉熵 = 真实分布的熵 + KL 散度"。在训练分类器时，真实分布 \(p\) 是固定的（由数据决定）——所以 \(H(p)\) 是常数；最小化交叉熵 \(H(p, q)\) 就等价于最小化 \(D_{KL}(p \| q)\)。换句话说：

训练分类器最小化交叉熵 = 把模型分布 \(q\) 推向真实分布 \(p\)。

这是一句话讲清楚了"为什么交叉熵损失被称为信息论上自然"。它不是工程师瞎选的——它就是 KL 散度的化身。

更有趣的：极大似然估计(MLE，第 17 章)和最小化交叉熵是同一件事。证明思路：MLE 最大化 \(\log L = \sum_i \log q_\theta(y_i)\)，等价于最小化 \(-\frac{1}{n} \sum_i \log q_\theta(y_i)\)；当 \(n \to \infty\)，这趋于 \(\mathbb{E}_{Y \sim p}[-\log q_\theta(Y)] = H(p, q_\theta)\)（交叉熵）。所以"最大化似然" = "最小化（经验上的）交叉熵" = "最小化模型与真实之间的 KL"。三者是同一回事的三种说法。

在二分类问题中，交叉熵简化成二元交叉熵（binary cross-entropy）：

\[ \mathrm{BCE} = -[y \log \hat y + (1 - y) \log (1 - \hat y)], \]

这是逻辑回归（第 22 章）的损失函数；多分类问题中变成 softmax + 交叉熵——你会在每个深度学习教程里见到。它们的根本根源都是 KL 散度。

另一个 ML 应用：变分推断（23.9 节简提过）的核心是最小化 \(D_{KL}(q \| p)\)（注意这是反方向的 KL，比 \(D_{KL}(p \| q)\) 更"模式搜寻"），用来近似复杂后验。

还有：互信息的应用——"信息瓶颈"理论用 \(I\) 量化网络层的"压缩"；图神经网络的对比学习用 \(I\) 当作目标函数；表示学习的"InfoNCE"损失就是互信息的下界。这条线索在现代 ML 里越来越主流。

本章到此结束。你见过了信息论的几位核心明星——自信息、熵、条件熵、互信息、KL 散度、信道容量——并看到它们如何统一了通信、压缩、机器学习这三件看似无关的事。Shannon 的杰作给我们留下了一套异常通用的语言，把"信息"作为一种可量化的物理量带进了科学。这门学科还有许多更深入的话题——率失真理论、网络信息论、量子信息——它们都建立在本章这几条朴素定义之上。
