第 16 章 区间估计与置信区间

上一章我们用最大似然给参数一个"最佳猜测" \(\hat\theta\)。可这种"一个数"的做法有它的不诚实之处——它从不告诉你"我有多确信"。两个完全不同的实验，一个抽 5 个数据、一个抽 5000 个数据，按 MLE 公式都能算出 \(\hat\theta\)；但前者明显应该比后者更"模糊"。一个数没法体现这种差别。

本章要做的事就是补上这个"模糊度"。具体方法是：不报告一个数，而报告一个区间，再附带一个数告诉你"这个区间有多大把握覆盖真值"。这就是置信区间（confidence interval, CI）。

本章我们会做四件事：

  • 把 CI 的定义从头讲明白，并把它最常被讲错的地方——"\(P(\theta \in \text{CI}) = 0.95\)" 这种错误解读——彻底纠正。
  • 构造正态总体均值的 CI（\(\sigma\) 已知和未知两种情形）。
  • 构造比例 \(p\)、方差 \(\sigma^2\) 的 CI，看到不同分布对应的 CI 长什么样。
  • 简单提一下大样本 CI 与 Bootstrap CI，让你知道 CI 是个"通用工具箱"，不只限于教科书几个案例。

16.1 一个数的局限

我们先用一个直白的例子让你"看到"为什么需要区间。

两个民调公司都在估总统支持率 \(p\)：

  • A 公司：抽样 \(n_A = 100\) 人，发现 60 人支持。\(\hat p_A = 0.60\)。
  • B 公司：抽样 \(n_B = 10000\) 人，发现 6000 人支持。\(\hat p_B = 0.60\)。

从"点估计"的角度，两家给出完全相同的答案：\(\hat p = 0.60\)。但你直觉上肯定会信 B 公司更多——它做了 100 倍的样本量。这种"信任度差异"在点估计里是看不见的。点估计把所有不确定性都扁平地压缩成"一个数"，损失掉了样本量、变异性等关键信息。

区间估计补上这个：

  • A 公司的 95% CI：\([0.504, 0.696]\)（宽度 \(\approx 0.19\)）。
  • B 公司的 95% CI：\([0.590, 0.610]\)（宽度 \(\approx 0.02\)）。

同一个 \(\hat p = 0.60\)，A 的区间是 B 的近 10 倍宽。"信谁更多"现在变成可量化的——这就是区间估计存在的核心理由。

这个 \(\sqrt n\) 收敛率在区间宽度上具体的表现：样本量乘 100，区间宽度除以 10。这就是 14.4 节那个"\(\sqrt n\)" 收敛率在用户层面看到的效果。

16.2 置信区间的形式定义

定义　设 \(\theta\) 是某未知参数，\(0 < \alpha < 1\)。一对随机统计量 \([L(\mathbf X), U(\mathbf X)]\)（其中 \(\mathbf X = (X_1, \ldots, X_n)\)）称为 \(\theta\) 的水平为 \(1 - \alpha\) 的置信区间，如果对每一个真实参数值 \(\theta\)，都成立

\[ \boxed{P_\theta(L(\mathbf X) \leq \theta \leq U(\mathbf X)) = 1 - \alpha.} \]

\(1 - \alpha\) 称为置信水平（confidence level）。常见取法：\(\alpha = 0.05\) 给 95% CI，\(\alpha = 0.01\) 给 99% CI，\(\alpha = 0.10\) 给 90% CI。

请你仔细看这条定义里"哪个是随机的"——括号里是 \(L(\mathbf X)\) 和 \(U(\mathbf X)\)，它们是数据的函数，是随机变量；而 \(\theta\) 是固定的、不变的真值（虽然我们不知道它具体是多少）。所以"\(L \leq \theta \leq U\)" 这个事件之所以有概率，是因为区间的端点在抖动，不是因为 \(\theta\) 在抖动。这是 CI 与贝叶斯"可信区间"最关键的区别——后者把 \(\theta\) 看成随机的。

这条"谁是随机的"问题非常重要，是下一节要正面处理的"CI 正确解读"的关键。

16.2.1 怎么造一个 CI：枢轴量法

在动手算具体的 CI 之前，介绍最常用的一招——枢轴量（pivot）法。

所谓"枢轴量"，是指一个函数 \(Q(\mathbf X, \theta)\)，它依赖样本和参数，但它的分布不依赖任何未知参数。最经典的例子：在 \(\mathcal{N}(\mu, \sigma_0^2)\) 总体下（\(\sigma_0\) 已知），

\[ Q(\mathbf X, \mu) = \frac{\bar X - \mu}{\sigma_0/\sqrt n} \sim \mathcal{N}(0, 1). \]

注意 \(Q\) 里既有数据又有 \(\mu\)，但它的分布是固定的标准正态，跟 \(\mu\) 一点关系都没有。这就是枢轴。

有了枢轴，构造 CI 就几乎机械化：选两个临界值 \(a, b\) 使 \(P(a \leq Q \leq b) = 1 - \alpha\)，再把"\(a \leq Q(\mathbf X, \theta) \leq b\)" 这个不等式对 \(\theta\) 反解，得到 \(L(\mathbf X) \leq \theta \leq U(\mathbf X)\)。这就是 CI 的端点。

下面几节我们就反复用这一招。

16.3 CI 的正确解读：被讲错最频繁的概率话题

这一节是本章最重要的一节。请你注意，因为统计学没有任何其他概念被讲错的频率比 CI 更高——连专业统计书里都常见解释失误。

设我们算出了一个具体的 95% CI \([0.504, 0.696]\)（数据已经具体了，端点已经是具体的数字）。下面是个无所不在的错误说法：

"真值 \(p\) 落在 \([0.504, 0.696]\) 里的概率是 95%。"

这句话错。原因深刻——一旦数据具体化了，端点是具体的数字（0.504 与 0.696），而 \(p\) 也是固定的真值（虽然我们不知道是多少）。两个具体的数字之间的"包含关系"，要么是（概率 1）要么不是（概率 0）——根本不存在 95% 的中间状态！这就好像问"7 落在区间 \([5, 9]\) 里的概率是多少"——这是个数学事实，不是概率事件。

那 95% 究竟是什么意思？正确的解读是：

"在我们用来构造这个区间的方法下，长期重复实验，95% 的区间会包含真值。"

详细些。想象你不是只做了一次实验，而是把整个实验重复一万次：每次抽 100 个人，每次按同样公式算出一个 CI。这一万次得到一万个不同的 CI——它们的端点都不同（因为每次的数据不同）。事实是：这一万个 CI 里大约有 9500 个会包含真值 \(p\)，500 个不会。

所以 95% 的把握度不是"这个特定区间包不包含 \(p\) 的概率"，而是"用这种构造方法生成的一族区间里，包含真值的比例"。这是一个关于构造方法的论断，不是关于这一次结果的论断。

这种解读称为"频率派"（frequentist）解读。频率派认为概率只能用于"长期重复实验的频率"，不能用于"对单一未知量的信念度"。

如果你想要"\(p\) 落在区间里的概率是 95%" 这种说法有意义，必须切换到贝叶斯派——把 \(p\) 也看成有先验分布的随机变量，看到数据后用贝叶斯公式更新。这种"贝叶斯可信区间"（credible interval）才有"参数落在里面的概率"的语义。但前提是你愿意承认"对参数赋一个先验分布"是合理的——这是另一个哲学问题，本书后面再讨论。

所以总结成一张口袋卡片：

  • 错误："\(P(\theta \in [0.504, 0.696]) = 0.95\)" ——把已经具体化的数当成随机变量。
  • 正确："如果重复抽样并每次按同样方法构造 CI，95% 的 CI 会包含 \(\theta\)。"
  • 口语版："我有 95% 的方法层面的把握，这个区间罩住了真值。"

把这三条记住，就基本免疫了 CI 的解读错误。

16.4 正态总体 \(\mu\) 的 CI（\(\sigma\) 已知）

从最简单的情形开始：总体 \(X \sim \mathcal{N}(\mu, \sigma^2)\)，\(\sigma\) 已知，\(\mu\) 未知。这种"已知 \(\sigma\)" 的情形在现实里很少见，但它是教科书的入门——后面所有 CI 都从它演化而来。

由 14.5 节，

\[ \frac{\bar X - \mu}{\sigma/\sqrt n} \sim \mathcal{N}(0, 1). \]

这就是我们的枢轴量。

为了得到 \(1 - \alpha\) 水平的 CI，找标准正态的双侧 \(\alpha\) 临界值 \(z_{\alpha/2}\)，使

\[ P(-z_{\alpha/2} \leq Z \leq z_{\alpha/2}) = 1 - \alpha. \]

常见值：\(z_{0.025} = 1.96\)（对应 95% CI）、\(z_{0.05} = 1.645\)（90% CI）、\(z_{0.005} = 2.576\)（99% CI）。

把枢轴量代入：

\[ P\!\left(-z_{\alpha/2} \leq \frac{\bar X - \mu}{\sigma/\sqrt n} \leq z_{\alpha/2}\right) = 1 - \alpha. \]

对 \(\mu\) 反解：

\[ P\!\left(\bar X - z_{\alpha/2}\frac{\sigma}{\sqrt n} \leq \mu \leq \bar X + z_{\alpha/2}\frac{\sigma}{\sqrt n}\right) = 1 - \alpha. \]

所以 \(\mu\) 的 \(1 - \alpha\) 水平 CI 是

\[ \boxed{\bar X \pm z_{\alpha/2} \cdot \frac{\sigma}{\sqrt n}.} \]

这个对称形式 \(\hat\theta \pm z_{\alpha/2} \cdot \mathrm{SE}\) 是所有正态分布 CI 的母板。请记住它的"骨架"：

(估计量) ± (临界值) × (标准误差)。

这条骨架后面要被复用很多次，每次只换具体的部件。

例 16.1（已知 \(\sigma\) 的均值 CI）　总体 \(\mathcal{N}(\mu, 100)\)（\(\sigma = 10\) 已知），抽 \(n = 25\)，得 \(\bar x = 47.3\)。给 \(\mu\) 的 95% CI。

解：\(z_{0.025} = 1.96\)，\(\sigma/\sqrt n = 10/5 = 2\)。

\[ \mathrm{CI} = 47.3 \pm 1.96 \times 2 = 47.3 \pm 3.92 = [43.38, 51.22]. \]

解读：用这个方法构造的区间，长期 95% 会包含真 \(\mu\)。

例 16.2（样本量与区间宽度）　如果想把 \(\mu\) 的 95% CI 宽度控制在 ±2 以内，最少要抽多少？

解：要 \(z_{0.025} \cdot \sigma/\sqrt n \leq 2\)，即 \(1.96 \times 10/\sqrt n \leq 2\)，\(\sqrt n \geq 9.8\)，\(n \geq 96.04\)，所以 \(n = 97\)。

这种"确定样本量"的反向问题在实验设计里非常常见。请记住一般公式：

\[ n \geq \left(\frac{z_{\alpha/2}\sigma}{E}\right)^2, \]

其中 \(E\) 是想要的"半宽度"（也叫边际误差，margin of error）。

16.5 正态总体 \(\mu\) 的 CI（\(\sigma\) 未知）

真实情形里 \(\sigma\) 通常也未知。这时怎么办？把 \(\sigma\) 替换成样本标准差 \(S\)，临界值从 \(z\) 换成 \(t\)。具体推导沿用 14.7 节：

\[ \frac{\bar X - \mu}{S/\sqrt n} \sim t_{n-1}. \]

用 \(t_{n-1, \alpha/2}\) 作临界值（自由度为 \(n - 1\) 的 \(t\) 分布的双侧 \(\alpha\) 临界），同样反解，得到

\[ \boxed{\bar X \pm t_{n-1, \alpha/2} \cdot \frac{S}{\sqrt n}.} \]

形式跟 16.4 节几乎一样，只是把 \((z, \sigma)\) 换成 \((t, S)\)。我们多了个未知参数 \(\sigma\)，付出的代价就是临界值变大、区间变宽。

请你注意，\(t_{n-1, \alpha/2}\) 比 \(z_{\alpha/2}\) 大（"胖尾"），所以 \(t\) 区间总比对应的 \(z\) 区间宽。这是合理的——我们对 \(\sigma\) 的额外不确定性，必然要让区间反映出来。\(n\) 大时 \(t \approx z\)，两种方法等价；\(n\) 小时差别明显。

例 16.3（\(\sigma\) 未知的 \(t\)-区间）　从某正态总体抽 \(n = 16\)，得 \(\bar x = 47.3\)、\(s = 9.5\)。给 \(\mu\) 的 95% CI。

解：自由度 \(n - 1 = 15\)，\(t_{15, 0.025} = 2.131\)（查表）。

\[ \mathrm{CI} = 47.3 \pm 2.131 \cdot 9.5/\sqrt{16} = 47.3 \pm 2.131 \cdot 2.375 = 47.3 \pm 5.06 = [42.24, 52.36]. \]

对比例 16.1（\(n = 25\)、\(\sigma\) 已知），那里得 \([43.38, 51.22]\)，宽度 7.84；这里宽度 10.12。两者样本量不一样不能完全直接比，但能感觉到"未知 \(\sigma\)" 让区间稍宽。

例 16.4（实测的小样本案例）　某药物的副作用持续时长（小时），抽 6 个病例：\(2.1, 4.5, 3.0, 6.2, 5.5, 3.7\)。假设近似正态。给 \(\mu\) 的 90% CI。

解：

\[ \bar x = (2.1 + 4.5 + 3.0 + 6.2 + 5.5 + 3.7)/6 = 25.0/6 \approx 4.167. \]

差值平方：\((2.1 - 4.167)^2 + \cdots = 4.27 + 0.11 + 1.36 + 4.13 + 1.78 + 0.22 = 11.87\)。

\[ s^2 = 11.87/(6-1) = 2.374, \quad s = 1.541. \]

自由度 5，\(t_{5, 0.05} = 2.015\)（90% CI 用 \(\alpha/2 = 0.05\)）。

\[ \mathrm{CI} = 4.167 \pm 2.015 \cdot 1.541/\sqrt 6 = 4.167 \pm 2.015 \cdot 0.629 = 4.167 \pm 1.268 = [2.90, 5.43]. \]

"我有 90% 的方法层面的把握，这个区间罩住了真实的副作用持续时长平均值。"

16.6 二项比例 \(p\) 的 CI

另一类极常见的推断是估"成功率"。设 \(X_1, \ldots, X_n \stackrel{\text{iid}}{\sim} \mathrm{Bernoulli}(p)\)，从样本 \(\hat p = \bar X = X/n\) 估 \(p\)。CI 怎么构造？

这件事比正态情形微妙——伯努利不是正态，但我们靠 CLT 来近似。

16.6.1 Wald CI（最朴素）

由 CLT，

\[ \frac{\hat p - p}{\sqrt{p(1-p)/n}} \xrightarrow{d} \mathcal{N}(0, 1). \]

但分母里有 \(p\)（未知），不能直接做枢轴。最朴素的修补：把分母里的 \(p\) 也用 \(\hat p\) 替换，得到近似的

\[ \frac{\hat p - p}{\sqrt{\hat p(1-\hat p)/n}} \approx \mathcal{N}(0, 1). \]

反解，得到 Wald CI：

\[ \boxed{\hat p \pm z_{\alpha/2}\sqrt{\frac{\hat p(1-\hat p)}{n}}.} \]

这是绝大多数教科书的"标准答案"，简单优雅。

但请你注意，Wald CI 在实践中表现不好——尤其当 \(\hat p\) 接近 0 或 1 时，区间会失真。比如 \(\hat p = 0\)（全部失败），公式给的 CI 是 \([0, 0]\)——一个零宽区间！这显然是错的，因为我们怎么可能"完全确信" \(p = 0\)？这种"边界塌陷"问题让 Wald CI 在小样本、极端比例时不可信。

16.6.2 Wilson CI（推荐）

1927 年 Edwin Wilson 提出的修正方法：不替换 \(p\)，而是直接对原不等式

\[ -z_{\alpha/2} \leq \frac{\hat p - p}{\sqrt{p(1-p)/n}} \leq z_{\alpha/2} \]

对 \(p\) 反解（这是个二次不等式）。这条不等式两边平方移项变成

\[ (\hat p - p)^2 \leq z_{\alpha/2}^2 \cdot \frac{p(1-p)}{n}, \]

关于 \(p\) 是个二次方程，求出两个根 \(L, U\) 即得 CI。展开后给出

\[ \boxed{\mathrm{Wilson \;CI} = \frac{\hat p + \dfrac{z^2}{2n}}{1 + \dfrac{z^2}{n}} \pm \frac{z}{1 + \dfrac{z^2}{n}}\sqrt{\dfrac{\hat p(1 - \hat p)}{n} + \dfrac{z^2}{4 n^2}},} \]

其中 \(z = z_{\alpha/2}\)。看起来复杂，但 Wilson CI 实践中覆盖率（实际包含真值的频率）比 Wald CI 接近名义水平得多，特别是小 \(n\) 或极端 \(p\) 的情形。Wilson 是当代统计软件（R、Python）的默认选择。

16.6.3 Agresti-Coull CI（折中）

2000 年 Agresti 与 Coull 提出一个非常简洁的"Wilson 近似"：把 Wald CI 用一个"伪样本"修正的 \(\hat p\) 算。具体定义

\[ \tilde n = n + z_{\alpha/2}^2, \qquad \tilde p = \frac{X + z_{\alpha/2}^2/2}{\tilde n}, \]

然后用 Wald 公式

\[ \tilde p \pm z_{\alpha/2}\sqrt{\frac{\tilde p(1-\tilde p)}{\tilde n}}. \]

对 95% CI，\(z^2 \approx 4\)，所以这等价于"假装多看到了 4 个观测：2 个成功、2 个失败"，再用朴素 Wald。这种"加 2 加 2"调整简单到能口算，覆盖率几乎跟 Wilson 一样好。

例 16.5（极端 \(\hat p\) 的对比）　某新疗法治了 \(n = 20\) 个病人，全部成功。\(\hat p = 1\)。给 95% CI。

解：

  • Wald CI：\(1 \pm 1.96\sqrt{1 \cdot 0/20} = 1 \pm 0 = [1, 1]\)。荒唐——只有 20 个病人就完全确信 \(p = 1\)？
  • Agresti-Coull：\(\tilde n = 20 + 4 = 24\)，\(\tilde p = (20 + 2)/24 \approx 0.917\)。CI \(= 0.917 \pm 1.96\sqrt{0.917 \cdot 0.083/24} \approx 0.917 \pm 0.110 = [0.807, 1.027]\)，截到 \([0.807, 1]\)。合理。
  • Wilson：算出 \([0.839, 1.000]\)。也合理。

这就是 Wald 在边界塌陷的活生生例子，请不要在 \(\hat p\) 接近 0 或 1 时用 Wald。

例 16.6（民调案例回到 16.1）　A 公司 \(n_A = 100\)、\(\hat p_A = 0.60\)；B 公司 \(n_B = 10000\)、\(\hat p_B = 0.60\)。算 95% Wald CI。

解：

  • A：\(0.60 \pm 1.96\sqrt{0.60 \cdot 0.40/100} = 0.60 \pm 1.96 \times 0.049 = 0.60 \pm 0.096 = [0.504, 0.696]\)。
  • B：\(0.60 \pm 1.96\sqrt{0.60 \cdot 0.40/10000} = 0.60 \pm 1.96 \times 0.0049 = 0.60 \pm 0.0096 = [0.590, 0.610]\)。

正是 16.1 节的两个区间。区别尽在 \(\sqrt n\) 倍——A 的区间是 B 的 10 倍宽。

16.7 方差 \(\sigma^2\) 的 CI（用 \(\chi^2\) 分布）

正态总体下，由 14.8 节，

\[ \frac{(n-1)S^2}{\sigma^2} \sim \chi^2_{n-1}. \]

这就是 \(\sigma^2\) 的枢轴。注意 \(\chi^2\) 分布不对称，所以 CI 端点要用非对称的临界值。

取 \(\chi^2_{n-1}\) 分布的两个分位数 \(\chi^2_{n-1, 1-\alpha/2}\)（下尾分位数）和 \(\chi^2_{n-1, \alpha/2}\)（上尾分位数），使

\[ P\!\left(\chi^2_{n-1, 1-\alpha/2} \leq \frac{(n-1)S^2}{\sigma^2} \leq \chi^2_{n-1, \alpha/2}\right) = 1 - \alpha. \]

对 \(\sigma^2\) 反解：

\[ \boxed{\frac{(n-1)S^2}{\chi^2_{n-1, \alpha/2}} \leq \sigma^2 \leq \frac{(n-1)S^2}{\chi^2_{n-1, 1-\alpha/2}}.} \]

请你注意这里大临界值 \(\chi^2_{n-1, \alpha/2}\) 落在下界的分母上，小临界值在上界的分母上——这是因为 "大除以大 = 小，小除以小 = 大"，反解出的不等式方向反过来了。如果不仔细可能下意识写错，请走两遍。

例 16.7（方差 CI）　从正态总体抽 \(n = 20\)，算得 \(s^2 = 14.5\)。给 \(\sigma^2\) 的 95% CI。

解：自由度 \(n - 1 = 19\)。查 \(\chi^2_{19}\) 表：\(\chi^2_{19, 0.025} = 32.852\)、\(\chi^2_{19, 0.975} = 8.907\)。所以

\[ \frac{19 \times 14.5}{32.852} \leq \sigma^2 \leq \frac{19 \times 14.5}{8.907}, \] \[ \frac{275.5}{32.852} \leq \sigma^2 \leq \frac{275.5}{8.907}, \] \[ 8.39 \leq \sigma^2 \leq 30.93. \]

这就是 \(\sigma^2\) 的 95% CI。注意这个区间非对称——\([\bar x - 6.11, \bar x + 16.43]\) 这种。原因就是 \(\chi^2\) 本身偏。如果想要 \(\sigma\) 的 CI，开平方两端：\(\sqrt{8.39} \leq \sigma \leq \sqrt{30.93}\)，即 \(2.90 \leq \sigma \leq 5.56\)（这一步合法是因为开平方是单调函数）。

16.8 大样本 CI：CLT 给的"通用近似"

16.4-16.7 都是正态总体下的精确 CI。但万一总体不是正态呢？比如总体是泊松、伽马、自由设的混合分布……怎么办？

答案：用大样本近似。

设 \(\hat\theta_n\) 是 \(\theta\) 的某个估计量（通常就是 MLE），它渐近正态：

\[ \sqrt n (\hat\theta_n - \theta) \xrightarrow{d} \mathcal{N}(0, V(\theta)). \]

这里 \(V(\theta)\) 是渐近方差（对 MLE 来说就是 \(1/I(\theta)\)）。如果我们把 \(V(\theta)\) 用 \(V(\hat\theta_n)\) 估计（这一步用 Slutsky 定理合法化），就得到

\[ \frac{\hat\theta_n - \theta}{\sqrt{V(\hat\theta_n)/n}} \xrightarrow{d} \mathcal{N}(0, 1). \]

然后构造大样本 CI

\[ \boxed{\hat\theta_n \pm z_{\alpha/2} \sqrt{V(\hat\theta_n)/n}.} \]

这就是几乎所有实际工作里 MLE 后给 CI 的标准做法。它的合理性来自 CLT 与 MLE 的渐近正态性，所以叫"大样本 CI"——\(n\) 要够大。这是个非常通用的工具。

例 16.8（泊松 \(\lambda\) 的大样本 CI）　某医院每天急诊人数近似 \(\mathrm{Poisson}(\lambda)\)。30 天观察，平均 \(\bar x = 12.4\)。给 \(\lambda\) 的 95% CI。

解：MLE \(\hat\lambda = \bar X = 12.4\)。泊松的 \(\mathrm{Var}(X) = \lambda\)，所以 \(\mathrm{Var}(\bar X) = \lambda/n\)。代入估计：

\[ \mathrm{CI} = 12.4 \pm 1.96\sqrt{12.4/30} = 12.4 \pm 1.96 \cdot 0.643 = 12.4 \pm 1.26 = [11.14, 13.66]. \]

注意我们没有用任何"正态总体"假设——总体是离散的泊松，CI 仍然能算。这就是大样本方法的威力。

16.9 CI 与假设检验的对偶

本节简短预告——CI 和下一章的假设检验，本质上是同一个东西的两个面。

详细说，假设检验是问"在指定的零假设 \(H_0: \theta = \theta_0\) 下，看到的数据有多极端？"，根据极端程度决定接受或拒绝 \(H_0\)。

而 CI 给一个区间 \([L, U]\)。两者的对偶关系是：

\[ \boxed{\theta_0 \in \text{CI}_{1 - \alpha} \;\Longleftrightarrow\; \text{在水平 } \alpha \text{ 下不拒绝 } H_0: \theta = \theta_0.} \]

也就是说：CI 是"所有不被数据拒绝的 \(\theta_0\) 值的集合"。把它倒过来想：你做 95% CI，等价于"用 5% 水平的检验测试每一个可能的 \(\theta_0\)，留下没被拒绝的那些"。

这条对偶非常深，意味着只要会做 CI 就会做检验，反过来也对。本章的所有 CI 公式，下章会被复用为检验公式。

16.10 Bootstrap CI：重抽样的革命

本章最后简短介绍一个 1979 年才被 Bradley Efron 发明的现代方法：Bootstrap。它和经典 CI 的关系，类似"现代雷达"与"传统三角测量"——做的事一样，但前者完全靠"硬算"，后者靠"理论公式"。

Bootstrap 的思想朴素得让人吃惊：

把样本本身当作总体的"代表"，从样本里反复有放回抽样模拟"重复实验"。

具体步骤：

  1. 给定原始样本 \(\mathbf x = (x_1, \ldots, x_n)\)。
  2. 从这 \(n\) 个数有放回地抽出 \(n\) 个，构成"自举样本" \(\mathbf x^*\)。
  3. 在 \(\mathbf x^*\) 上算 \(\hat\theta^* = \hat\theta(\mathbf x^*)\)。
  4. 重复 1-3 共 \(B\) 次（\(B\) 通常取 1000、10000 量级），得到 \(B\) 个 \(\hat\theta^*\) 值。
  5. 这 \(B\) 个 \(\hat\theta^*\) 值的分布近似就是 \(\hat\theta\) 的抽样分布。直接取其 \(\alpha/2\) 与 \(1 - \alpha/2\) 分位数当 CI 端点（这是"百分位 Bootstrap CI"）。

这件事的合理性来自一条简单观察：原样本 \(\hat F_n\)（经验分布函数）是真实 \(F\) 的好近似（Glivenko-Cantelli 定理）。所以"从 \(\hat F_n\) 里再抽样"近似于"从真 \(F\) 里再抽样"——后者就是抽样分布的来源。

Bootstrap 相对于经典 CI 的优势：

  • 通用：对任何统计量都能做——中位数、相关系数、机器学习模型的预测……不需要"找一个枢轴 + 推它的分布"。
  • 不靠正态假设：完全靠数据本身的分布。
  • 计算友好：在现代计算机上几乎不要钱。

劣势：

  • 对小样本（\(n < 20\)）效果一般——经验分布太"颗粒"。
  • 对极端分位数估计不稳。
  • 当数据有强相关时（时间序列、空间数据）需要专门的"块 Bootstrap"等修正。

例 16.9（Bootstrap 的工作机制）　从某未知总体抽 \(n = 6\)：\(2.1, 4.5, 3.0, 6.2, 5.5, 3.7\)（与例 16.4 同）。想给样本中位数（不是均值！）的 95% CI。

解：经典理论里"中位数的 CI"公式不简单（要查顺序统计量的分布表）。Bootstrap 直接：

  1. 从这 6 个数有放回抽 6 个，比如 \((2.1, 4.5, 3.0, 4.5, 3.7, 2.1)\)，算中位数 \(\hat m^* = 3.35\)。
  2. 再抽，再算……重复 \(B = 10000\) 次。
  3. 10000 个 \(\hat m^*\) 排序，取第 250 个和第 9750 个分别当下、上界，得 95% Bootstrap CI。

结果可能是 \([2.4, 5.5]\) 这样的区间——具体数字依赖每次抽样的随机性，但相互间会很接近。

这种"自举"的奇妙之处在于：你没有对总体分布做任何假设，没有需要任何理论分布表，但还是构造出了一个像模像样的 CI。这就是计算时代统计学的代表方法。

实际工作中两种方法并用：经典 CI 在数学上漂亮、在标准设定（正态/二项/泊松）下精确；Bootstrap 在复杂模型、非标准统计量、机器学习场景里几乎是默认工具。"两个工具都会用" 是当代统计工作者的基本素养。

16.11 本章总结：CI 的"骨架公式"

把本章所有 CI 收成一张骨架表。它们形式高度统一，请看出共性：

  • 正态 \(\mu\)（\(\sigma\) 已知）：\(\bar X \pm z_{\alpha/2}\sigma/\sqrt n\)。
  • 正态 \(\mu\)（\(\sigma\) 未知）：\(\bar X \pm t_{n-1, \alpha/2} S/\sqrt n\)。
  • 二项 \(p\)（Wald）：\(\hat p \pm z_{\alpha/2}\sqrt{\hat p(1-\hat p)/n}\)。
  • 正态 \(\sigma^2\)：\(\dfrac{(n-1)S^2}{\chi^2_{n-1, \alpha/2}} \leq \sigma^2 \leq \dfrac{(n-1)S^2}{\chi^2_{n-1, 1-\alpha/2}}\)。
  • 大样本通用：\(\hat\theta_n \pm z_{\alpha/2}\sqrt{\hat V/n}\)。

共性是 (估计量) ± (临界值) × (标准误差)。临界值从分布表查；标准误差是估计量本身的"抖动幅度"，由抽样分布给。

请你最后再注意一遍："CI 是关于构造方法的把握度"，不是"参数落在区间里的概率"。这件事请在脑子里牢牢钉死，将来你的统计直觉不会出错。

下一章我们把同样的"枢轴量 + 临界值"工具反过来用——不是"给一个区间"，而是"判一个具体的 \(\theta_0\) 是否被数据排斥"。这就是假设检验，统计学最常用、也最常被误用的方法。区间估计的对偶就在那里。
